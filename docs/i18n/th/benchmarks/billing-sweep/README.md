# Anthropic vision-billing sweep

🌐 แปลแล้ว: [ทุกภาษา](../../../README.md)

**เหตุผลที่มีอยู่:** เกทด้านความสามารถในการทำกำไรจะปลอดภัยได้ก็ต่อเมื่อ
การประมาณต้นทุน *เป๊ะ* เท่านั้น สูตรที่คลาดเคลื่อนไปเพียงเล็กน้อยจะทำให้
บล็อกที่จริงแล้วมีต้นทุนสูงกว่าถูกแปลงไปด้วย ดังนั้นการ sweep นี้จึงยึดสูตร
กับตัวเลขจริงของ API ก่อนนำไปใช้งานจริง — ให้เหลือ **ค่าคลาดเคลื่อนเป็นศูนย์**

```
what the sweep decides, visually:

  patch model     ⌈w/28⌉ × ⌈h/28⌉ + overhead        ← current docs
  retired /750    (w · h) / 750                       ← old formula
                       │
                       ▼  probe geometries chosen to separate the two by 25–180 tokens/row
  measured 1568×728 page = 1,460 tokens
     patch predicts 1,456  ✅   (residual ~0)
     /750  predicts 1,522  ✗   (off by 62)
```

การ sweep แบบฟรีด้วย `count_tokens` ที่ตัดสินคำถามด้านรูปทรงที่ยังเปิดอยู่สองข้อ:

1. **สูตร** — API คิดค่าใช้จ่ายเป็นแพตช์ `ceil(w/28) × ceil(h/28)` (เอกสารปัจจุบัน)
   หรือสูตรที่เลิกใช้แล้ว `w·h/750`? ชุดการตรวจสอบแยกทั้งสองออกจากกันด้วยผลต่าง 25–180
   โทเคนต่อแถว
2. **ระดับ** — `claude-fable-5` ได้รับขีดจำกัดระดับความละเอียดสูงหรือไม่ (ขอบยาว
   ≤ 2576 px, ≤ 4784 visual token)? แถว `page-old-1928x1928` เป็นตัว
   ชี้ขาด: ≈ **4761** ที่วัดได้ หมายความว่าระดับความละเอียดสูงเป็น WYSIWYG (หน้าใหญ่แบบเก่า
   บรรจุตัวอักษรได้มากกว่าหน้า 1568×728 ปัจจุบันประมาณ 3.3 เท่า ที่อัตราส่วนตัวอักษร/โทเคนเท่ากัน);
   ≈ **1521** หมายความว่าเป็นการ resample ในระดับมาตรฐาน และ 1568×728 ยังคงถูกต้อง

บริบท: การ sweep เมื่อ 2026-07-01 ที่อยู่เบื้องหลังหน้า 1568×728 ปัจจุบัน
(การตรวจสอบความสามารถในการอ่าน, 2026-07-01) วัดผล
บน `claude-sonnet-4-5` — โมเดลระดับมาตรฐาน — ในขณะที่การใช้งานจริงเล็งไปที่
Fable 5 ซึ่งเอกสารด้านการมองเห็นจัดไว้ในระดับความละเอียดสูง การตรวจสอบนั้น
ยังวัดหน้าปัจจุบันได้ 1460 โทเคน: ใกล้เคียงกับสูตรแพตช์ 1456 มากกว่า
1522 ของ /750 บ่งชี้ว่า API ได้เปลี่ยนไปใช้การคิดค่าใช้จ่ายแบบแพตช์แล้ว

## การรัน

```bash
pnpm run build                              # dist/ prerequisite (like all evals)
node benchmarks/billing-sweep/run.mjs --dry-run   # predictions only, no key, $0

ANTHROPIC_API_KEY=sk-... node benchmarks/billing-sweep/run.mjs \
  --models claude-fable-5,claude-sonnet-4-5 --probe-multi --probe-20plus
```

ต้องยิงตรงไปที่ API **โดยตรง** — ห้ามผ่านพร็อกซี OmniGlyph เด็ดขาด เพราะจะแปลง
body `count_tokens` ใช้ฟรี; การ sweep แบบเต็มทำประมาณ 25 คำขอ

## การอ่านผลลัพธ์

ในแต่ละโมเดล แต่ละแถวการตรวจสอบแสดงจำนวนโทเคนภาพที่วัดได้ (มีภาพลบด้วย
baseline แบบข้อความล้วน) เทียบกับการคาดการณ์ทั้งสี่แบบ (`patch`/`legacy750` ×
`standard`/`highres`); สรุปจัดอันดับสมมติฐานตามค่าคลาดเคลื่อนสัมบูรณ์เฉลี่ย
`--probe-multi` ตรวจสอบขีดจำกัดต่อภาพ (2×1092² ≈ 2×1521); `--probe-20plus`
ตรวจสอบกฎ >20 ภาพ (ด้านที่ยาวกว่า 2000 px ต้องถูกปฏิเสธ ไม่ใช่ resample)
แถวจะไปอยู่ใน `results/*.jsonl`; สูตรการคาดการณ์อยู่ใน `formulas.mjs`
ถูกยึดไว้โดย `tests/billing-sweep-formulas.test.ts`
