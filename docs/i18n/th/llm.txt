# OmniGlyph

> พร็อกซีบีบอัดบริบทในรูปแบบภาพสำหรับ LLM พร็อกซีในเครื่องที่เขียนส่วนที่
> ใหญ่ของแต่ละคำขอ (system prompt, เอกสารเครื่องมือ, ประวัติเก่า, ผลลัพธ์
> เครื่องมือขนาดใหญ่) ใหม่เป็นหน้า PNG แบบ 1-bit ความหนาแน่นสูง ก่อนที่จะออกจากเครื่อง
> โดยใช้การคำนวณค่าใช้จ่ายแบบเป๊ะต่อผู้ให้บริการ (Anthropic/OpenAI/Gemini) เพื่อให้
> แปลงเฉพาะเมื่อการคำนวณโทเคนคุ้มค่า การลดค่าใช้จ่ายแบบครบวงจรที่วัดได้:
> 59–70% ส่วนหนึ่งของตระกูล OmniRoute; ยังจัดส่งเป็นเอนจินบีบอัด `omniglyph`
> ภายใน OmniRoute ด้วย

**คุณค่าหลัก:** ชี้เครื่องมือใดๆ ที่รองรับ Anthropic ไปที่
`ANTHROPIC_BASE_URL=http://127.0.0.1:47821` แล้วบริบทขนาดใหญ่จะถูกคิดค่าใช้จ่ายเป็น
โทเคนภาพ (~19 ตัวอักษร/โทเคน) แทนที่จะเป็นโทเคนข้อความ (~2 ตัวอักษร/โทเคน)

**เวอร์ชันปัจจุบัน:** 1.0.0 · **สัญญาอนุญาต:** MIT · **รันไทม์:** Node ≥18 และ
Cloudflare Workers

## วิธีการทำงาน

1. เกทตรวจสอบความคุ้มค่า: คำนวณค่าใช้จ่ายภาพที่เป๊ะของผู้ให้บริการ
   (Anthropic: ⌈w/28⌉ × ⌈h/28⌉ + ค่าใช้จ่ายเสริมต่อบล็อก, วัดได้ค่าคลาดเคลื่อนเป็นศูนย์)
   และแปลงเฉพาะเมื่อประหยัดโทเคนได้จริง
2. เกทโมเดล (fail-closed): มีเพียงโมเดลที่ผ่าน benchmark การอ่านเท่านั้นที่
   จะได้รับภาพ ได้รับอนุมัติในตอนนี้: Claude Fable 5 (อ่านแบบเป๊ะ 100%)
   ถูกบล็อกพร้อมหลักฐาน: GPT-5.5 (0/60), Gemini 2.5-flash (กุเรื่อง),
   Opus 4.8 อยู่ในโหมดปลอดภัยแบบเลือกเปิดใช้เท่านั้น (กลิฟขนาด 4 เท่า)
3. เรนเดอร์: reflow + atlas กลิฟ 1-bit 5×8 → หน้า PNG มาตรฐาน 1568×728
   (ระดับความละเอียดสูงเป็นกับดักด้านค่าใช้จ่ายที่วัดผลแล้ว — ไม่ได้ใช้งาน)
4. นำหน้ากลับเข้าที่เดิมแบบเป็นมิตรกับแคช; ค่าที่ต้องเป๊ะทุกไบต์ (hash/ID) เดินทาง
   ควบคู่ไปเป็นข้อความ; การอ่านที่ล้มเหลวจะงดตอบ (ไม่มีการกุเรื่องแบบเงียบใน
   ~300 การทดสอบ)
5. ลองใหม่เมื่อถูกปฏิเสธ: ถ้าโมเดลปฏิเสธหน้าที่เรนเดอร์แล้ว คำขอเดิม
   จะถูกเล่นซ้ำโดยอัตโนมัติ

## โครงสร้างโปรเจกต์

- `src/core/` — pipeline การแปลง, การคำนวณค่าใช้จ่ายแบบเป๊ะ, ตัวเรนเดอร์, แคชการเรนเดอร์แบบ
  LRU, พร็อกซีที่มีการลองใหม่เมื่อถูกปฏิเสธ การคำนวณค่าใช้จ่ายถูกส่งออกที่รากของแพ็กเกจ
- `src/node.ts` — โฮสต์ในเครื่องพร้อมแดชบอร์ด (โทเคนที่ประหยัดได้, การแปลง
  แบบเทียบเคียงกัน, สวิตช์ปิดฉุกเฉิน) `src/worker.ts` — โฮสต์ Cloudflare Workers
- `benchmarks/` — ชุดทดสอบ billing-sweep และ density-frontier; ทุกตัวเลขที่
  เผยแพร่มีหลักฐาน JSONL ใน `benchmarks/*/results/`
- `docs/` — architecture/, benchmarks/ (ระเบียบวิธี + ผลลัพธ์), ops/
  (คู่มือ OmniRoute), i18n/pt-BR/

## บันทึกสำคัญสำหรับ LLM

- TDD แบบเข้มงวดและการวัดผลก่อนการกล่าวอ้าง: ตัวเลขที่ไม่มีหลักฐานจาก
  benchmark จะไม่ได้รับการยอมรับ; เกท fail-closed ต้องไม่ถูกลดความเข้มงวดเพื่อ
  ทำให้การทดสอบผ่าน
- การบีบอัดเกิดขึ้นที่ฝั่งคำขอเท่านั้น — ผลลัพธ์จากโมเดลสตรีมผ่านไปโดยไม่ถูกแตะต้อง
- พร็อกซีทำงานในเครื่องเป็นหลัก: ไม่มีอะไรถูกส่งออกไปยังบุคคลที่สามเพิ่มเติม
