## แบบจำลองภัยคุกคาม (OmniGlyph คืออะไร)

OmniGlyph เป็น **พร็อกซีในเครื่อง** ระหว่างไคลเอนต์ของคุณ (เช่น Claude Code) และ
API ของ LLM ตามการออกแบบ มันมองเห็นเนื้อหาเซสชันทั้งหมดของคุณและข้อมูลรับรองของคุณระหว่าง
การส่งผ่าน การตัดสินใจด้านความปลอดภัยที่เกี่ยวข้อง:

- **ผูกกับ loopback เป็นค่าเริ่มต้น** (`127.0.0.1`): แดชบอร์ดไม่มีการ
  ยืนยันตัวตนและให้บริการบริบทเซสชันที่ถูกจับไว้ (ข้อความต้นฉบับของ
  ภาพ, ข้อมูลทางไกล) `HOST=0.0.0.0` เป็นการเปิดใช้งานที่ต้องเลือกเองอย่างชัดเจนและเปิดเผยทั้งหมด
  นั้นสู่เครือข่าย — ใช้เฉพาะบนเครือข่ายที่เชื่อถือได้เท่านั้น
- **ข้อมูลรับรอง**: พร็อกซีส่งต่อ auth header ของไคลเอนต์ไปยัง
  upstream และไม่เก็บไว้ คีย์ที่ให้มาผ่าน env
  (`ANTHROPIC_API_KEY` เป็นต้น) จะอยู่ในหน่วยความจำเท่านั้น
- **ข้อมูลทางไกลในเครื่อง**: `~/.omniglyph/events.jsonl` เก็บข้อมูลเมตาต่อคำขอ
  (จำนวนโทเคน, hash ของเนื้อหา) และ เมื่อเกิด error 4xx จะเก็บตัวอย่างเนื้อหาแบบบีบอัด —
  ให้ปฏิบัติต่อไฟล์นี้ว่ามีความอ่อนไหว
- **เนื้อหาที่ถูกแปลงเป็นภาพเป็นแบบ lossy**: ค่าที่ต้องเป๊ะทุกไบต์ (secret, hash) ต้องไม่
  ขึ้นอยู่กับการอ่านภาพ pipeline เก็บค่าเหล่านั้นไว้เป็นข้อความ แต่กฎทองคือ
  อย่าใส่ secret ไว้ในบริบทของ LLM
- **ซัพพลายเชน**: `pnpm-workspace.yaml` บังคับใช้ `minimumReleaseAge` เป็น
  3 วันสำหรับแพ็กเกจใหม่ทุกตัว; แกนหลักมี dependency รันไทม์เพียงตัวเดียว
