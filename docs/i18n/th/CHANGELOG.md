# บันทึกการเปลี่ยนแปลง

รูปแบบ: [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) · semantic versioning

## [1.0.0] — 2026-07-07

การเปิดตัวสู่สาธารณะครั้งแรก

### ตัวผลิตภัณฑ์

- **พร็อกซีบีบอัดบริบทในรูปแบบภาพ**: เขียนส่วนที่ใหญ่ของแต่ละคำขอ LLM
  (system prompt, เอกสารเครื่องมือ, ประวัติเก่า, ผลลัพธ์เครื่องมือขนาดใหญ่) ใหม่เป็นหน้า PNG
  แบบ 1-bit ความหนาแน่นสูง ก่อนที่จะออกจากเครื่องของคุณ มีเซิร์ฟเวอร์ Node ในเครื่อง และโฮสต์
  Cloudflare Workers
- **คำนวณค่าใช้จ่ายแบบเป๊ะต่อผู้ให้บริการ** (`src/core/`): แพตช์ Anthropic ขนาด 28px +
  ค่าใช้จ่ายเสริม 3–4 โทเคน/บล็อก (จากการตรวจสอบของเราเอง ค่าคลาดเคลื่อนเป็นศูนย์), สูตรของ OpenAI
  และ Gemini ตรวจสอบเทียบกับเอกสารทางการ ส่งออกที่รากของแพ็กเกจ
  (`anthropicImageTokens`, `resolveAnthropicVisionTier`, ขีดจำกัดของแต่ละระดับ)
- **ค่าคอนฟิกการเรนเดอร์สำหรับใช้งานจริงที่วัดผลแล้ว**: atlas กลิฟ 1-bit ความหนาแน่นสูง (ไม่มี
  anti-aliasing), หน้าระดับมาตรฐาน — ทุกทางเลือกมีหลักฐานจาก benchmark รองรับใน
  `benchmarks/*/results/`
- **ชุดทดสอบ Benchmark** (`benchmarks/`): billing-sweep (การนับโทเคน) และ
  density-frontier (ขอบเขตความแม่นยำในการอ่านข้ามโมเดล/ความหนาแน่น) รันซ้ำได้
  ผ่าน API, OpenRouter, Claude Code CLI หรือผ่าน OmniRoute
  (`--via-omniroute`)
- **การลองใหม่เมื่อถูกปฏิเสธ**: ตัวตรวจจับ SSE/JSON เล่นคำขอเดิมซ้ำเมื่อโมเดล
  ปฏิเสธหน้าที่เรนเดอร์แล้ว (สวิตช์ปิดฉุกเฉิน `retryRefusalWithOriginal`)
- **แคชการเรนเดอร์แบบ LRU** สำหรับหน้าที่กำหนดผลลัพธ์แน่นอน
- **เอนจิน OmniRoute**: จัดส่งเป็นเอนจินบีบอัด `omniglyph` ใน
  [OmniRoute](https://github.com/diegosouzapw/OmniRoute) (โหมดเดี่ยวและ
  pipeline แบบเรียงซ้อน) พร้อมเกท fail-closed และการนับโทเคนที่รับรู้ภาพ

### ตัวเลข (ทำซ้ำได้ทั้งหมด)

- ตัวอย่างเรนเดอร์ UI: 1015 ตัวอักษร → PNG 438×120, 254 → 84 โทเคน (**ประหยัด 66.9%**)
- หน้ามาตรฐาน 1568×728 = 1456 โทเคนภาพ ไม่ว่าจะบรรจุข้อความมากแค่ไหนก็ตาม
- Claude อ่านหน้า 1-bit ความหนาแน่นสูงได้ 100% ที่ความหนาแน่นระดับใช้งานจริง; Opus 4.8 อ่านได้
  77–87% ที่ 10×16

### การตัดสินใจเชิงลบ (จากการวัดผล ไม่ใช่ความเห็นส่วนตัว)

- **ระดับความละเอียดสูงเป็นกับดักด้านค่าใช้จ่าย**: หน้า 1928² ถูกคิดค่าใช้จ่ายแบบ WYSIWYG
  แต่ encoder ไม่ได้รับความละเอียดเต็ม — ทั้งสองระดับเรนเดอร์เป็นหน้ามาตรฐาน
- **GPT-5.5 ถูกปฏิเสธ**: อ่านแถบความหนาแน่นสูงได้ 0/60 ครั้ง และขยายการตอบกลับ
  ~40 เท่าเทียบกับกลุ่มควบคุมที่เป็นข้อความ
- **gpt-4o-mini ไม่เคยถูกแปลงเป็นภาพ** (ขีดจำกัดโทเคน 2833/5667 ทำให้ไม่คุ้มค่า)
- **Gemini 2.5-flash กุเรื่อง** แทนที่จะงดตอบบนหน้าความหนาแน่นสูง
  (0/26) — รอทดสอบซ้ำด้วยโควตาแบบเสียเงิน
