## 1. Anthropic 计费(直接调用 count_tokens,$0,11 种几何结构 × 2 个模型)

已确认的公式:按档位缩放后 `tokens = ceil(w/28) × ceil(h/28)`,
**每块 +3(Fable 5)/ +4(Sonnet 4.5)**——所有行的残差均为零。

| 探针 | 尺寸 | Fable 5(高分辨率) | Sonnet 4.5(标准) |
|---|---|---:|---:|
| 文档锚点 | 1092×1092 | 1524 | 1525 |
| 文档锚点 | 1000×1000 | 1299 | 1300 |
| 标准页面 | 1568×728 | 1459 | 1460 |
| **大页面** | **1928×1928** | **4764(所见即所得!)** | 1525(重采样) |
| 高分辨率上限 | 1960×1960 | 4764(截断) | 1525 |
| 高分辨率长边 | 2576×1204 | 3959 | 1516 |
| 窄高条带 | 768×1932 | 1935 | 1292(重采样) |
| 2×1092²(多图) | — | 3048(= 2×1524) | 3049 |
| 21×280²(超过 20 张图) | — | 2163 | 2164 |
| 20×280²+2100² | — | 6824(2100² → 缩小处理,在 count_tokens 中并未被拒绝) | 3585 |

由此得出的决策(已实现):精确的按分块门控;按模型区分档位
(Fable/Mythos/Opus 4.8/4.7/Sonnet 5 = 高分辨率);`cols` 由 313 改为 312。

## 2. 阅读准确率(density-frontier,十六进制/camelCase/数字探针 + 干扰项)

|---|---:|---:|---:|
| 标准 1568×728 · **1-bit** | **30/30(100%)** | 0 | 0 |
| 标准 1568×728 · AA | 25/30(83%) | 5 | 0 |

| 配置 | 精确 | 弃权 | 错误 |

## 补充 2026-07-06 —— 经由直连 API 的 A/B 测试(165 次调用):H1/H2 被证伪

| 配置 | 精确 | 弃权 | 拒绝 | 错误 |
|---|---:|---:|---:|---:|
| fable 标准 5×8(AA 与 1-bit) | 0/60 | 0 | **60/60 拒绝** | 0 |
| fable 高分辨率 5×8 AA | 2/30 | 21 | 3 | 4(其中 4 次被预测到) |
| fable 高分辨率 5×8 1-bit | 1/30 | 23 | 1 | 3(其中 2 次被预测到) |
| opus 高分辨率 10×16 | **23/30(77%)** | 1 | 0 | 6 |

结论:高分辨率档位的 1928² 页面在计费上确实是所见即所得(测得 4764 token,
扫描测试确认),但**编码器并未获得完整分辨率**——只有 1-2/30 能读取,
且出现单字形替换类错误(6→8、a→4),这是内部重采样的典型特征。
**计费 ≠ 编码器实际输入 → 陷阱:成本高 3.3 倍,可读性反而更差。**
已应用:pageGeometryForTier() 已回退——两个档位都渲染 1568×728;
档位相关基础设施予以保留(精确计费依然有效,未来重新调优只需改 1 行代码)。
H3 更新:"传输层重采样"(也)是 API 自身编码器所为。原始 API 上转录类问题的
拒绝率:标准页面上为 100%(H4 得到强化——只有智能体式措辞能规避)。
Opus 10×16 在两种传输方式上均得到确认(77-87%)。

## 补充 2026-07-06(3)—— Gemini 2.5-flash(部分测试:免费档位配额在运行中耗尽)

在配额耗尽前跑通的约 26 个图像回答中:**0 次正确,1 次弃权,约 25 次虚构**——
而且这些不是字形混淆:它们是随机数字(`indexLedgerInd → 0040375615`),
也就是说编码器在被测密度下几乎什么都看不到(原生瓦片 42 字符/token,
以及 MEDIUM 固定成本),而 2.5-flash 会**编造答案**而非弃权
(无视 ILEGIVEL 指令)。文本对照组:在跑通的问题中为 3/3。没有输出膨胀
(每次回答 6-28 token)。

初步信号:H5/H6 在 2.5-flash 上倾向于"否",且其失败模式**比 GPT 更糟糕**
(静默虚构而非弃权)——若要接入 Gemini,代理需要额外的防护措施。
待关闭事项:用付费配额或换一天重新运行,并测试 gemini-2.5-pro
(flash 是该系列中读取能力最弱的)。原生瓦片页面仍然拥有已记录的最佳比率
(42.3 字符/token);存疑的是可读性本身。

成本说明:部分页面(语料的最后一页)在瓦片计费模式下计费不划算
(高度较小 → 裁剪单元较小 → 瓦片数更多)——若要接入 Gemini,
将最后一页的高度补齐到 1152px 是一项必须实施的优化。
