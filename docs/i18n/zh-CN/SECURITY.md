# 安全策略

## 报告漏洞

请在 GitHub 上开启一个私密的安全公告
(<https://github.com/diegosouzapw/OmniGlyph/security/advisories/new>),
或直接联系维护者(diegosouza.pw@outlook.com)。请勿为可利用的漏洞开启公开议题。

## 威胁模型(OmniGlyph 是什么)

OmniGlyph 是介于你的客户端(例如 Claude Code)与 LLM API 之间的**本地代理**。
按设计,它能看到你所有的会话内容以及传输中的凭据。相应的安全决策如下:

- **默认绑定到回环地址**(`127.0.0.1`):仪表盘没有身份验证,会展示捕获到的
  会话上下文(图像的源文本、遥测数据)。`HOST=0.0.0.0` 是需要显式开启的选项,
  会将这一切暴露给网络——只应在受信任的网络中使用。
- **凭据**:代理会将客户端的鉴权请求头转发给上游,且不会持久化它们。
  通过环境变量提供的密钥(如 `ANTHROPIC_API_KEY` 等)仅保留在内存中。
- **本地遥测**:`~/.omniglyph/events.jsonl` 保存每次请求的元数据
  (token 计数、请求体哈希),以及在 4xx 错误时的压缩请求体样本——
  应将该文件视为敏感文件。
- **图像化的内容是有损的**:字节精确的值(密钥、哈希)绝不能依赖于图像读取;
  流水线会将它们保留为文本,但黄金法则始终是:不要把密钥放进 LLM 上下文里。
- **供应链**:`pnpm-workspace.yaml` 为任何新增依赖包强制设置了 3 天的
  `minimumReleaseAge`;核心库只有一个运行时依赖。

## 支持的版本

只有最新的发布线(`main` / 最新的 `v1.x`)会获得修复。
