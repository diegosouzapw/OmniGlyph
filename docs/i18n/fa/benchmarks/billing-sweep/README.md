# بررسی صورت‌حساب بینایی Anthropic

بررسی رایگان `count_tokens` که دو پرسش باز دربارهٔ هندسه را تعیین می‌کند:

1. **فرمول** — آیا API پچ‌های `ceil(w/28) × ceil(h/28)` (مستندات فعلی) را
   صورت‌حساب می‌کند یا `w·h/750` بازنشسته‌شده را؟ مجموعهٔ پروب این دو را با
   ۲۵ تا ۱۸۰ توکن به‌ازای هر ردیف تفکیک می‌کند.
2. **لایه** — آیا `claude-fable-5` سقف‌های وضوح‌بالا را دریافت می‌کند (لبهٔ
   بلند ≤ 2576 پیکسل، ≤ 4784 توکن بصری)؟ ردیف `page-old-1928x1928` تصمیم‌گیرنده
   است: حدود **۴۷۶۱** اندازه‌گیری‌شده یعنی WYSIWYG وضوح‌بالا (صفحهٔ بزرگ
   قدیمی حدود ۳.۳ برابر کاراکتر بیشتر به‌ازای هر تصویر دارد نسبت به 1568×728
   امروزی، با همان کاراکتر بر توکن)؛ حدود **۱۵۲۱** یعنی تغییر نمونهٔ لایهٔ
   استاندارد، و 1568×728 صحیح می‌ماند.

زمینه: بررسی 2026-07-01 پشت صفحهٔ فعلی 1568×728 (حسابرسی خوانایی،
2026-07-01) روی `claude-sonnet-4-5` — یک مدل لایهٔ استاندارد — اندازه‌گیری
شد، در حالی که تولید هدفش Fable 5 است، که مستندات بینایی آن را در لایهٔ
وضوح‌بالا قرار می‌دهند. آن حسابرسی همچنین صفحهٔ فعلی را در 1460 توکن
اندازه‌گیری کرد: نزدیک‌تر به 1456 فرمول پچ نسبت به 1522 مربوط به /750،
که نشان می‌دهد API از قبل به صورت‌حساب پچ منتقل شده بود.

## اجرا

```bash
pnpm run build                              # پیش‌نیاز dist/ (مانند همهٔ evalها)
node benchmarks/billing-sweep/run.mjs --dry-run   # فقط پیش‌بینی‌ها، بدون کلید، $0

ANTHROPIC_API_KEY=sk-... node benchmarks/billing-sweep/run.mjs \
  --models claude-fable-5,claude-sonnet-4-5 --probe-multi --probe-20plus
```

باید **مستقیماً** به API برخورد کند — هرگز از طریق پراکسی OmniGlyph، که
بدنه را تبدیل می‌کند. `count_tokens` رایگان است؛ بررسی کامل حدود ۲۵
درخواست می‌سازد.

## خواندن خروجی

به‌ازای هر مدل، هر ردیف پروب توکن‌های تصویر اندازه‌گیری‌شده را (با تصویر
منهای خط پایهٔ فقط-متن) در برابر هر چهار پیش‌بینی
(`patch`/`legacy750` × `standard`/`highres`) نشان می‌دهد؛ خلاصه فرضیه‌ها را
بر اساس میانگین باقی‌ماندهٔ مطلق رتبه‌بندی می‌کند.
`--probe-multi` سقف به‌ازای هر تصویر را بررسی می‌کند (2×1092² ≈ 2×1521)؛
`--probe-20plus` قانون بیش از ۲۰ تصویر را بررسی می‌کند (یک لبهٔ بیش از
2000 پیکسل باید رد شود، نه تغییر نمونه). ردیف‌ها در `results/*.jsonl`
قرار می‌گیرند؛ ریاضیات پیش‌بینی در `formulas.mjs` است، پین‌شده توسط
`tests/billing-sweep-formulas.test.ts`.
