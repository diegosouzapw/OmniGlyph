# بنچمارک‌ها

هر عددی که OmniGlyph ادعا می‌کند از یکی از دو مجموعهٔ زیر می‌آید —
قابل اجرای مجدد، تا حد امکان قطعی، با رسیدهای خام هر پاسخ در
`*/results/*.jsonl`. تحلیل تلفیقی: [docs/benchmarks/BENCHMARKS.md](../docs/benchmarks/BENCHMARKS.md).

## ۱. `billing-sweep/` — یک تصویر واقعاً چقدر هزینه دارد؟

پروب‌های رایگان `count_tokens` در برابر API زندهٔ Anthropic، که فرمول
بازنشسته‌شدهٔ `w·h/750` را با مدل پچ ۲۸ پیکسلی فعلی در سراسر ۱۱ هندسهٔ پروب
روی ۲ مدل × ۲ لایهٔ وضوح مقایسه می‌کند.

**نتیجه (2026-07-05): مدل پچ با باقی‌ماندهٔ صفر روی هر پروب برازش می‌شود**
— صورت‌حساب = `⌈w/28⌉ × ⌈h/28⌉` پس از تغییر اندازهٔ لایه، به‌علاوهٔ +۳/+۴
توکن ثابت به‌ازای هر بلاک تصویر. صفحهٔ تولیدی (1568×728) دقیقاً 1,460 توکن
هزینه دارد و 28,080 کاراکتر حمل می‌کند ≈ **۱۹.۲ کاراکتر بر توکن** در برابر
حدود ۲ کاراکتر بر توکن برای متن فشرده.

```bash
node benchmarks/billing-sweep/run.mjs --dry-run          # فقط پیش‌بینی‌ها، $0
ANTHROPIC_API_KEY=... node benchmarks/billing-sweep/run.mjs   # بررسی زنده، همچنان $0 (count_tokens رایگان است)
```

## ۲. `density-frontier/` — آیا مدل واقعاً می‌تواند آن را بخواند؟

هزینه (آفلاین، دقیق) × دقت خواندن (زنده) در سراسر پیکربندی‌های رندر،
هندسهٔ صفحه، اطلس‌های گلیف و ارائه‌دهنده‌ها. پیکره needleهای رشته‌ای دقیق
(شناسه‌های hex، camelCase، دنباله‌های رقمی) به‌همراه **distractorهای
نزدیک-به-هم ساخته‌شده از جفت‌های گیج‌کنندگی گلیف اندازه‌گیری‌شده** را
می‌کارد — بنابراین ساختهٔ خیالی بی‌صدا تشخیص داده می‌شود، نه فقط اشتباه
شمرده می‌شود. امتیازدهی قطعی است (بدون داور LLM):
`correct` / `abstained` (`ILEGIVEL` صادقانه) / `silent_wrong` / `no_answer`.

**نتایج اصلی** (n=30 به‌ازای هر بازو):

| بازو | خواندن دقیق | یادداشت |
|---|---:|---|
| Fable 5 · صفحهٔ استاندارد · اطلس ۱-بیت (تولید) | **30/30** | صفر خطا، صفر ساختهٔ خیالی |
| Fable 5 · صفحهٔ استاندارد · اطلس AA (پیش‌فرض قدیمی) | 25/30 | ۵ امتناع صادقانه — چرا تولید به ۱-بیت تغییر کرد |
| Fable 5 · صفحهٔ وضوح‌بالا 1928² | 1–2/30 | ۳.۳ برابر صورت‌حساب می‌شود اما رمزگذار تغییر نمونه می‌دهد — تلهٔ صورت‌حساب، فعال نشده |
| Opus 4.8 · گلیف‌های 10×16 | 23–26/30 | حالت ایمن اختیاری |
| GPT-5.5 · نوار 768px (هر دو اطلس) | 0/60 | + حدود ۴۰ برابر تورم توکن خروجی در برابر کنترل متنی خودش (30/30، 62 توکن) |
| Gemini 2.5-flash (جزئی، سهمیه) | 0/26 | به‌جای امتناع، ابداع می‌کند |

سه لایهٔ انتقال: API مستقیم (`ANTHROPIC_API_KEY`/`OPENAI_API_KEY`/`GEMINI_API_KEY`)،
OpenRouter (`OPENROUTER_API_KEY`)، و `--via-cli` (اشتراک Claude Code —
$0). نکتهٔ آموخته‌شده به‌سختی: واسط‌ها (OpenRouter، ابزار Read در CLI)
تصاویر بزرگ را تغییر نمونه می‌دهند؛ فقط نتایج API مستقیم برای خوانایی
معتبرند.

```bash
pnpm exec tsx benchmarks/density-frontier/run.ts --dry-run                    # جدول هزینه، $0
pnpm exec tsx benchmarks/density-frontier/run.ts --via-cli --sections 30     # از طریق اشتراک، $0
ANTHROPIC_API_KEY=... pnpm exec tsx benchmarks/density-frontier/run.ts --configs anthropic-std-5x8-1bit
```

تست‌های واحد که بخش‌های خالص را پین می‌کنند (پیکره، امتیازدهی، فرمول‌های
هزینه): `tests/billing-sweep-formulas.test.ts`،
`tests/density-frontier.test.ts`، `tests/anthropic-vision.test.ts`،
`tests/gemini-profiles.test.ts`، `tests/gpt-billing-audit.test.ts`.
