# Sweep за таксуване на визия при Anthropic

🌐 Translated: [all languages](../../../README.md)

**Защо съществува:** вратата за печалба е безопасна само ако оценката на цената е
*точна*. Формула, отклонена дори малко, би преобразувала блокове, които всъщност
струват повече. Затова този sweep закотвя формулата към реалните числа на API-то преди да
влезе в производство — до **нулев остатък**.

```
what the sweep decides, visually:

  patch model     ⌈w/28⌉ × ⌈h/28⌉ + overhead        ← current docs
  retired /750    (w · h) / 750                       ← old formula
                       │
                       ▼  probe geometries chosen to separate the two by 25–180 tokens/row
  measured 1568×728 page = 1,460 tokens
     patch predicts 1,456  ✅   (residual ~0)
     /750  predicts 1,522  ✗   (off by 62)
```

Безплатен sweep с `count_tokens`, който решава два отворени въпроса за геометрията:

1. **Формула** — таксува ли API-то `ceil(w/28) × ceil(h/28)` пача (текуща
   документация) или изоставената `w·h/750`? Наборът от проби разделя двете с 25–180
   токена на ред.
2. **Ниво** — получава ли `claude-fable-5` лимитите за висока резолюция (дълъг ръб
   ≤ 2576 px, ≤ 4784 визуални токена)? Редът `page-old-1928x1928` е
   решаващият: ≈ **4761** измерено означава WYSIWYG на високо ниво (старата голяма страница носи
   ~3.3× повече символи на изображение от днешната 1568×728, при същите символи/токен);
   ≈ **1521** означава пренасемплиране на стандартното ниво, и 1568×728 остава коректно.

Контекст: sweep-ът от 2026-07-01 зад текущата страница 1568×728
(одит на четимостта, 2026-07-01) е измерен
на `claude-sonnet-4-5` — модел от стандартно ниво — докато производството цели
Fable 5, който документацията за визия поставя в нивото с висока резолюция. Този одит
също измери текущата страница на 1460 токена: по-близо до 1456 на формулата с пачове,
отколкото до 1522 на /750, намеквайки, че API-то вече е преминало към таксуване по пачове.

## Изпълнение

```bash
pnpm run build                              # dist/ prerequisite (like all evals)
node benchmarks/billing-sweep/run.mjs --dry-run   # predictions only, no key, $0

ANTHROPIC_API_KEY=sk-... node benchmarks/billing-sweep/run.mjs \
  --models claude-fable-5,claude-sonnet-4-5 --probe-multi --probe-20plus
```

Трябва да удари API-то **директно** — никога през OmniGlyph проксито, което би
трансформирало тялото. `count_tokens` е безплатен; пълният sweep прави ~25 заявки.

## Четене на изхода

На модел, всеки ред на проба показва измерените токени за изображение (с изображение минус
базовата линия само за текст) спрямо всичките четири прогнози (`patch`/`legacy750` ×
`standard`/`highres`); резюмето класира хипотезите по средна абсолютна остатъчна стойност.
`--probe-multi` проверява лимита на изображение (2×1092² ≈ 2×1521); `--probe-20plus`
проверява правилото за >20 изображения (страна >2000 px трябва да бъде отхвърлена, не преоразмерена).
Редовете попадат в `results/*.jsonl`; математиката на прогнозите живее в `formulas.mjs`,
закотвена от `tests/billing-sweep-formulas.test.ts`.

## След присъдата

- Потвърдена формула с пачове → портиране на OmniGlyph PR #27 (точен превод на преоразмеряване) и
  съгласуване на математиката на вратата `ANTHROPIC_PIXELS_PER_TOKEN` в `src/core/transform.ts`.
- Потвърдено ниво с висока резолюция при Fable → повторно въвеждане на геометрия на страница по ниво
  (страници от клас 1928×1928 за Fable/Opus 4.8/Sonnet 5, 1568×728 за стандартно),
  отразявайки как пътят на GPT вече поддържа собствения си `GPT_MAX_HEIGHT_PX`.
