# Бенчмаркове

Всяко число, което OmniGlyph твърди, идва от един от двата harness-а по-долу —
повторно изпълними, детерминистични, където е възможно, със сурови доказателства по отговор в
`*/results/*.jsonl`. Консолидиран анализ: [docs/benchmarks/BENCHMARKS.md](../docs/benchmarks/BENCHMARKS.md).

## 1. `billing-sweep/` — колко всъщност струва изображение?

Безплатни `count_tokens` проби срещу живия Anthropic API, сравняващи изоставената
формула `w·h/750` срещу текущия модел с 28 px пач-размери в 11 геометрии на проба
на 2 модела × 2 нива на резолюция.

**Резултат (2026-07-05): моделът с пачове съвпада с остатък НУЛА при всяка проба**
— таксувано = `⌈w/28⌉ × ⌈h/28⌉` след преоразмеряване по ниво, плюс фиксирани +3/+4 токена на
блок от изображение. Производствената страница (1568×728) струва точно 1460 токена и
носи 28 080 символа ≈ **19.2 символа/токен** срещу ~2 символа/токен като плътен текст.

```bash
node benchmarks/billing-sweep/run.mjs --dry-run          # само прогнози, $0
ANTHROPIC_API_KEY=... node benchmarks/billing-sweep/run.mjs   # живо измерване, все още $0 (count_tokens е безплатен)
```

## 2. `density-frontier/` — може ли моделът наистина да ГО ПРОЧЕТЕ?

Цена (офлайн, точна) × точност на четене (на живо) в различни конфигурации за рендиране, геометрии
на страницата, атласи с глифи и доставчици. Корпусът засажда точни низови needles (hex ID-та,
camelCase, поредици от цифри) плюс **разсейващи елементи, близки до истината, изградени от
измерените двойки на объркваемост на глифите** — така че тихата конфабулация се засича, не просто
брои като грешна. Оценяването е детерминистично (без LLM-съдия):
`correct` / `abstained` (честно `ILEGIVEL`) / `silent_wrong` / `no_answer`.

**Основни резултати** (n=30 на рамо):

| рамо | точни четения | бележки |
|---|---:|---|
| Fable 5 · стандартна страница · атлас 1-бит (производство) | **30/30** | нула грешки, нула конфабулация |
| Fable 5 · стандартна страница · AA атлас (стар по подразбиране) | 25/30 | 5 честни въздържания — защо производството премина на 1-бит |
| Fable 5 · страница с висока резолюция 1928² | 1–2/30 | таксувана 3.3× повече, но енкодерът пренасемплира — капанът при таксуването, не активиран |
| Opus 4.8 · глифи 10×16 | 23–26/30 | безопасният режим по избор |
| GPT-5.5 · лента 768px (и двата атласа) | 0/60 | + ~40× надуване на изходните токени спрямо собствения му текстов контрол (30/30, 62 токена) |
| Gemini 2.5-flash (частично, квота) | 0/26 | конфабулира вместо да се въздържа |

Три транспорта: директен API (`ANTHROPIC_API_KEY`/`OPENAI_API_KEY`/`GEMINI_API_KEY`),
OpenRouter (`OPENROUTER_API_KEY`), и `--via-cli` (абонамент за Claude Code —
$0). Урок, научен по трудния начин: посредниците (OpenRouter, инструментът Read на CLI-то)
пренасемплират големи изображения; само резултатите от директния API са авторитетни за
четимостта.

```bash
pnpm exec tsx benchmarks/density-frontier/run.ts --dry-run                    # таблица с разходи, $0
pnpm exec tsx benchmarks/density-frontier/run.ts --via-cli --sections 30     # чрез абонамент, $0
ANTHROPIC_API_KEY=... pnpm exec tsx benchmarks/density-frontier/run.ts --configs anthropic-std-5x8-1bit
```
