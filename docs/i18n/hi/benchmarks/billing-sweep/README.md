# Anthropic विज़न-बिलिंग स्वीप

🌐 अनुवादित: [सभी भाषाएँ](../../../README.md)

**यह क्यों मौजूद है:** प्रॉफ़िटेबिलिटी गेट तभी सुरक्षित है जब लागत अनुमान
*सटीक* हो। थोड़ा भी चूका हुआ फ़ॉर्मूला ऐसे ब्लॉकों को बदल देगा जिनकी असल
लागत ज़्यादा होती है। इसलिए यह स्वीप शिप होने से पहले फ़ॉर्मूले को API के
वास्तविक आँकड़ों पर पिन करता है — **शून्य अवशिष्ट** तक।

```
what the sweep decides, visually:

  patch model     ⌈w/28⌉ × ⌈h/28⌉ + overhead        ← current docs
  retired /750    (w · h) / 750                       ← old formula
                       │
                       ▼  probe geometries chosen to separate the two by 25–180 tokens/row
  measured 1568×728 page = 1,460 tokens
     patch predicts 1,456  ✅   (residual ~0)
     /750  predicts 1,522  ✗   (off by 62)
```

मुफ़्त `count_tokens` स्वीप जो दो खुले ज्यामिति प्रश्नों का फ़ैसला करता है:

1. **फ़ॉर्मूला** — क्या API `ceil(w/28) × ceil(h/28)` पैच का बिल लगाता है
   (वर्तमान दस्तावेज़) या सेवानिवृत्त `w·h/750` का? प्रोब सेट इन दोनों को
   प्रति पंक्ति 25–180 टोकन से अलग करता है।
2. **टियर** — क्या `claude-fable-5` को हाई-रिज़ॉल्यूशन कैप मिलती है (लंबा
   किनारा ≤ 2576 px, ≤ 4784 दृश्य टोकन)? `page-old-1928x1928` पंक्ति
   निर्णायक है: ≈ **4761** मापा गया मतलब है हाई-रेस WYSIWYG (पुराना बड़ा
   पेज आज के 1568×728 की तुलना में प्रति छवि ~3.3× अधिक अक्षर रखता है,
   उसी chars/token पर); ≈ **1521** का मतलब है स्टैंडर्ड-टियर resample,
   और 1568×728 सही बना रहता है।

संदर्भ: वर्तमान 1568×728 पेज के पीछे 2026-07-01 का स्वीप (पठनीयता
ऑडिट, 2026-07-01) `claude-sonnet-4-5` पर मापा गया था — एक स्टैंडर्ड-टियर
मॉडल — जबकि प्रोडक्शन Fable 5 को लक्षित करता है, जिसे विज़न दस्तावेज़
हाई-रिज़ॉल्यूशन टियर में रखते हैं। उस ऑडिट ने वर्तमान पेज को भी 1460
टोकन पर मापा: /750 के 1522 की तुलना में पैच फ़ॉर्मूले के 1456 के अधिक
क़रीब, जो संकेत देता है कि API पहले ही पैच बिलिंग में जा चुका था।

## चलाएँ

```bash
pnpm run build                              # dist/ prerequisite (like all evals)
node benchmarks/billing-sweep/run.mjs --dry-run   # predictions only, no key, $0

ANTHROPIC_API_KEY=sk-... node benchmarks/billing-sweep/run.mjs \
  --models claude-fable-5,claude-sonnet-4-5 --probe-multi --probe-20plus
```

API को **सीधे** मारना ज़रूरी है — कभी भी OmniGlyph प्रॉक्सी के ज़रिए नहीं,
जो बॉडी को बदल देगा। `count_tokens` मुफ़्त है; पूरा स्वीप ~25 अनुरोध
करता है।

## आउटपुट पढ़ना

प्रति मॉडल, हर प्रोब पंक्ति मापे गए छवि टोकन दिखाती है (छवि-सहित माइनस
केवल-टेक्स्ट बेसलाइन) सभी चार भविष्यवाणियों (`patch`/`legacy750` ×
`standard`/`highres`) के विरुद्ध; सारांश परिकल्पनाओं को माध्य निरपेक्ष
अवशिष्ट के अनुसार क्रम देता है। `--probe-multi` प्रति-छवि कैप जाँचता है
(2×1092² ≈ 2×1521); `--probe-20plus` >20-छवि नियम जाँचता है (एक >2000 px
पक्ष को अस्वीकार किया जाना चाहिए, resample नहीं)। पंक्तियाँ
`results/*.jsonl` में जाती हैं; भविष्यवाणी गणित `formulas.mjs` में रहता
है, जिसे `tests/billing-sweep-formulas.test.ts` द्वारा पिन किया गया है।
