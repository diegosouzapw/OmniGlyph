# المساهمة في OmniGlyph

شكرًا لاهتمامك! هذا المشروع لديه قاعدتان ثقافيتان غير قابلتين للتفاوض —
وهما السبب في أن كل رقم في الـ README يمكن الوثوق به.

## القاعدة 1 — TDD صارم

كل كود إنتاجي يولد من اختبار فشل أولًا:

1. اكتب الاختبار و**شاهده يفشل للسبب الصحيح**.
2. اكتب الحد الأدنى لجعله ينجح.
3. أعد الهيكلة (refactor) مع البقاء أخضر (نجاح الاختبارات).

الشرط الكامل هو: `pnpm run typecheck && pnpm test && pnpm run build` — الثلاثة
معًا، دائمًا (فحص روابط المستندات وحارس إعادة التسمية يعملان داخل `pnpm test`
عبر `tests/docs-integrity.test.ts`).

## القاعدة 2 — القياس قبل الادّعاء

لا يُدمج أي تغيير على الهندسة، أو الأطلس، أو معادلة الفوترة، أو نطاق النماذج
بدون رقم مقيس. المستودع مبني حول هذا الانضباط:

- تكلفة الفوترة ← أثبتها بـ `benchmarks/billing-sweep/` (`count_tokens`
  مجاني؛ الانحراف المتوقع: صفر).
- الوضوح (legibility) ← أثبته بـ `benchmarks/density-frontier/` (n≥30 لكل ذراع،
  تقييم حتمي، إيصالات JSONL مُدمجة في `benchmarks/*/results/`).
- شرط قبول تغيير إعداد افتراضي في الإنتاج: جوهر == ضابط النص
  **و** صفر أخطاء صامتة في السلاسل الدقيقة **و** وفورات إيجابية.

الفرضيات بلا أرقام تذهب إلى `docs/ROADMAP.md` كفرضيات — أبدًا إلى الـ README
كحقائق. فكرتان "بديهيتان" فُنّدتا بالفعل بالبيانات
(الصفحة عالية الدقة، الأطلس بتنعيم الحواف)؛ العملية تعمل.

## الإعداد

```bash
pnpm install
pnpm test              # الحزمة الكاملة، ~40–90 ثانية
pnpm run dev:node      # بروكسي محلي في وضع المراقبة (watch)
```

Node ≥18 (يختبر CI الإصدارات 20/22/24)، pnpm 10 (مثبَّت بواسطة `packageManager` في
package.json).

## البنية

| المجلد | القاعدة |
|---|---|
| `src/core/` | مستقل عن بيئة التشغيل (Web APIs فقط — يعمل على Node وWorkers) |
| `src/node.ts` / `src/worker.ts` | توصيلات المضيف فقط |
| `benchmarks/` | حزم قابلة لإعادة التشغيل؛ نتائج JSONL هي إيصالات، مُدمجة |
| `docs/` | benchmarks/ (الأرقام)، architecture/ (الخريطة)، ROADMAP (الفرضيات)، ops/ (OmniRoute) |

## الالتزامات (Commits) وطلبات السحب (PRs)

- التزامات تقليدية (`feat:`، `fix:`، `perf:`، `docs:`، `refactor:`،
  `test:`، `chore:`)، مع نص يشرح *السبب* مع الأرقام ذات الصلة.
- طلبات سحب صغيرة ومركّزة؛ تغييرات السلوك تأتي مع الاختبار الذي يثبّتها، وعند
  الاقتضاء، مع القياس الذي يبرّرها.
- لا تعِد كتابة كتل `cache_control` الخاصة بالعميل، ولا تضِف تبعيات تشغيل
  بدون نقاش (النواة خفيفة التبعيات عن قصد)، ولا تستخدم
  `Math.random`/الطوابع الزمنية في مسارات الرندر (الحتمية قيد صارم،
  مُختبَر بتطابق البايتات).
