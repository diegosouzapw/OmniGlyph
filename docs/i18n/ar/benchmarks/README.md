# المقاييس (Benchmarks)

🌐 مُترجَم: [كل اللغات](../../README.md)

كل رقم يدّعيه OmniGlyph يأتي من إحدى الحزمتين أدناه —
قابلة لإعادة التشغيل، حتمية حيثما أمكن، مع إيصالات خام لكل إجابة في
`*/results/*.jsonl`. التحليل الموحَّد: [docs/benchmarks/BENCHMARKS.md](../docs/benchmarks/BENCHMARKS.md).

## كيف تعمل الوفورات (في صورة واحدة)

يحاسب المزوّدون **النص بالرمز (token)**، لكن يحاسبون **الصورة بأبعادها** —
لا بمقدار النص المحشور بداخلها. صفحة قياسية واحدة تكلفة ثابتة بغضّ النظر
عن كثافة النص:

```
                         ┌─────────────────────────────────────────────┐
28,080 characters   →    │  one 1568 × 728 PNG page   =  1,460 tokens   │
of dense context         └─────────────────────────────────────────────┘
                                          the SAME 1,460 whether the page
                                          holds 200 chars or 28,080
```

نفس السياق، محاسَب بطريقتين:

```
as dense TEXT    ██████████████████████████████████████████████  ~14,040 tokens
as ONE IMAGE     █████                                              1,460 tokens
                                                                    ▼
                                                              ~10× fewer tokens
```

لماذا تفوز الصورة — عدد الأحرف المحمولة لكل رمز:

```
image (dense page)  ███████████████████████████████████████  19.2 chars/token
dense text          ████                                       ~2  chars/token
```

لا يقوم OmniGlyph بهذا التبديل إلا عندما تقول الرياضيات الدقيقة إنه يفوز، وفقط
للنماذج المُثبَت أنها تقرأ الصفحة. الحزمتان أدناه تُثبتان كل نصف من هذه المعادلة.

## 1. `billing-sweep/` — كم تكلّف الصورة فعليًا؟

عيّنات `count_tokens` المجانية مقابل API الحيّ لـ Anthropic، تقارن
معادلة `w·h/750` المتقاعدة بنموذج رقاقة 28 بكسل الحالي عبر 11 هندسة عيّنة
على نموذجين × مستويي دقة.

**النتيجة (2026-07-05): نموذج الرقاقة يتطابق بانحراف صفري على كل عيّنة**
— المبلغ المُحاسَب = `⌈w/28⌉ × ⌈h/28⌉` بعد إعادة الحجم حسب المستوى،
بالإضافة إلى +3/+4 رموز ثابتة لكل كتلة صورة. الصفحة الإنتاجية (1568×728) تكلّف بالضبط 1,460 رمزًا
وتحمل 28,080 حرفًا ≈ **19.2 حرفًا/رمز** مقابل ~2 حرف/رمز كنص كثيف.

```
the page as the patch grid the API actually bills:

  ⌈1568 / 28⌉ = 56 patches wide
  ⌈ 728 / 28⌉ = 26 patches tall
  ───────────────────────────────
  56 × 26  = 1,456  + 4 per-block  =  1,460 tokens   (flat, WYSIWYG)
```

```bash
node benchmarks/billing-sweep/run.mjs --dry-run          # predictions only, $0
ANTHROPIC_API_KEY=... node benchmarks/billing-sweep/run.mjs   # live sweep, still $0 (count_tokens is free)
```

## 2. `density-frontier/` — هل يستطيع النموذج قراءتها فعليًا؟

التكلفة (دون اتصال، دقيقة) × دقة القراءة (حيّة) عبر إعدادات الرندر، هندسات
الصفحات، أطالس الرموز، والمزوّدين. تزرع المجموعة ألغازًا بسلاسل دقيقة حرفيًا
(معرّفات hex، camelCase، سلاسل أرقام) بالإضافة إلى **مشتِّتات قريبة الشبه مبنية
من أزواج الخلط المقيسة للرموز** — بحيث يُكتشف الاختلاق الصامت، لا أن
يُحسب خطأً فحسب. التقييم حتمي (بلا حكم LLM):
`correct` / `abstained` (`ILEGIVEL` صادقة) / `silent_wrong` / `no_answer`.

**النتائج الرئيسية** (n=30 لكل ذراع):

| الذراع | قراءات دقيقة | ملاحظات |
|---|---:|---|
| Fable 5 · صفحة قياسية · أطلس أحادي البت (الإنتاج) | **30/30** | صفر أخطاء، صفر اختلاق |
| Fable 5 · صفحة قياسية · أطلس AA (افتراضي قديم) | 25/30 | 5 امتناعات صادقة — لماذا انتقل الإنتاج إلى أحادي البت |
| Fable 5 · صفحة عالية الدقة 1928² | 1–2/30 | تُحاسَب 3.3× لكن المُرمِّز يعيد أخذ عينات — فخّ الفوترة، غير مُفعَّل |
| Opus 4.8 · رموز 10×16 | 23–26/30 | الوضع الآمن الاختياري |
| GPT-5.5 · شريط 768px (كلا الأطلسين) | 0/60 | + تضخّم مخرجات ~40× مقابل ضابط النص الخاص به (30/30، 62 رمزًا) |
| Gemini 2.5-flash (جزئي، حصة) | 0/26 | يختلق بدل الامتناع |

دقة القراءة بلمحة — هذه هي بوّابة النموذج ذات الفشل المغلق نفسها، مرسومة:

```
Fable 5 · 1-bit page (prod)   ██████████████████████████████  30/30  ✅ approved
Opus 4.8 · 10×16 (safe mode)  ████████████████████████░░░░░░  ~24/30 ⚠️ opt-in
Fable 5 · high-res 1928²      █░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  ~2/30  🚫 billing trap
GPT-5.5 · 768px strip         ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  0/60   ⛔ blocked
Gemini 2.5-flash              ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  0/26   ⛔ confabulates
```

فقط الذراع ✅ يُشحَن إنتاجيًا. أي شيء يقرأ بشكل سيّئ يُحظَر *مع إيصال*، ونظام
التقييم الثلاثي يعني أن النموذج الذي يخمّن خطأً (`silent_wrong`) يُعامَل كأسوأ
من نموذج يمتنع بصدق (`ILEGIVEL`).

ثلاث طبقات نقل: API مباشر (`ANTHROPIC_API_KEY`/`OPENAI_API_KEY`/`GEMINI_API_KEY`)،
OpenRouter (`OPENROUTER_API_KEY`)، و`--via-cli` (اشتراك Claude Code —
$0). تحذير مُتعلَّم بالطريقة الصعبة: الوسطاء (OpenRouter، أداة Read الخاصة
بـ CLI) يعيدون أخذ عينات للصور الكبيرة؛ فقط نتائج API المباشر موثوقة
للوضوح.

```bash
pnpm exec tsx benchmarks/density-frontier/run.ts --dry-run                    # cost table, $0
pnpm exec tsx benchmarks/density-frontier/run.ts --via-cli --sections 30     # via subscription, $0
ANTHROPIC_API_KEY=... pnpm exec tsx benchmarks/density-frontier/run.ts --configs anthropic-std-5x8-1bit
```
