# مسح فوترة الرؤية لـ Anthropic

🌐 مُترجَم: [كل اللغات](../../../README.md)

**سبب وجوده:** بوّابة الربحية آمنة فقط إذا كان تقدير التكلفة *دقيقًا*. معادلة
منحرفة ولو قليلًا كانت ستحوّل كتلًا تكلّف فعليًا أكثر. لذا يثبّت هذا المسح
المعادلة على الأرقام الحقيقية للـ API قبل الشحن — إلى **انحراف صفري**.

```
what the sweep decides, visually:

  patch model     ⌈w/28⌉ × ⌈h/28⌉ + overhead        ← current docs
  retired /750    (w · h) / 750                       ← old formula
                       │
                       ▼  probe geometries chosen to separate the two by 25–180 tokens/row
  measured 1568×728 page = 1,460 tokens
     patch predicts 1,456  ✅   (residual ~0)
     /750  predicts 1,522  ✗   (off by 62)
```

مسح `count_tokens` المجاني الذي يحسم سؤالي هندسة مفتوحين:

1. **المعادلة** — هل تحاسب الـ API رقاقات `ceil(w/28) × ceil(h/28)` (المستندات
   الحالية) أم `w·h/750` المتقاعدة؟ مجموعة العيّنات تفصل بين الاثنتين بفارق
   25–180 رمزًا لكل صف.
2. **المستوى** — هل يحصل `claude-fable-5` على حدود الدقة العالية (الحافة الطويلة
   ≤ 2576 بكسل، ≤ 4784 رمزًا بصريًا)؟ صف `page-old-1928x1928` هو
   الحاسم: ≈ **4761** مقيسة تعني WYSIWYG لمستوى الدقة العالية (الصفحة الكبيرة
   القديمة تحمل ~3.3× أحرفًا أكثر لكل صورة من صفحة 1568×728 اليوم، بنفس نسبة
   حرف/رمز)؛ ≈ **1521** تعني إعادة أخذ عينات بالمستوى القياسي، وتبقى 1568×728 صحيحة.

السياق: المسح بتاريخ 2026-07-01 خلف صفحة 1568×728 الحالية
(تدقيق الوضوح، 2026-07-01) قِيس
على `claude-sonnet-4-5` — نموذج مستوى قياسي — بينما الإنتاج يستهدف
Fable 5، الذي تضعه مستندات الرؤية في مستوى الدقة العالية. ذلك التدقيق
قاس أيضًا الصفحة الحالية عند 1460 رمزًا: أقرب إلى معادلة الرقاقة 1456 من
معادلة /750 البالغة 1522، مما يوحي بأن الـ API انتقلت بالفعل إلى فوترة الرقاقة.

## التشغيل

```bash
pnpm run build                              # dist/ prerequisite (like all evals)
node benchmarks/billing-sweep/run.mjs --dry-run   # predictions only, no key, $0

ANTHROPIC_API_KEY=sk-... node benchmarks/billing-sweep/run.mjs \
  --models claude-fable-5,claude-sonnet-4-5 --probe-multi --probe-20plus
```

يجب أن يصل إلى الـ API **مباشرة** — أبدًا عبر بروكسي OmniGlyph، الذي سيحوّل
المتن. `count_tokens` مجاني؛ المسح الكامل يجري ~25 طلبًا.

## قراءة المخرجات

لكل نموذج، يعرض كل صف عيّنة رموز الصورة المقيسة (مع الصورة ناقص أساس النص فقط)
مقابل التنبؤات الأربعة (`patch`/`legacy750` × `standard`/`highres`)؛ يرتّب
الملخص الفرضيات حسب متوسط الانحراف المطلق. `--probe-multi` يتحقق من الحد
الأقصى لكل صورة (2×1092² ≈ 2×1521)؛ `--probe-20plus` يتحقق من قاعدة أكثر من
20 صورة (ضلع أكبر من 2000 بكسل يجب أن يُرفض، لا أن يُعاد تحجيمه).
تُسجَّل الصفوف في `results/*.jsonl`؛ رياضيات التنبؤ تعيش في `formulas.mjs`،
مثبَّتة بـ `tests/billing-sweep-formulas.test.ts`.
