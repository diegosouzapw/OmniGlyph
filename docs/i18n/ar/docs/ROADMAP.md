## المرحلة 1 — إصلاحات فوترة متعدّدة المزوّدين (أخطاء مؤكدة في التدقيق)

الأولوية محددة بالتدقيق (مستندات رسمية موثقة بتاريخ 2026-07-05):

1. ☐ **D2 (بوّابة مقلوبة)**: `gpt-4o-mini` يقع في البلاطة الافتراضية 85/170، لكن تكلفته **2833 أساسي / 5667 لكل بلاطة** (~33× أقل من التقدير، ~0.8 حرف/رمز) — تصويره خسارة دائمًا والبوّابة توافق عليه. `src/core/gpt-model-profiles.ts:51-59`.

3. ☐ **D1**: مضاعِف `o4-mini` 1.62 ← **1.72** (يُقدَّر بأقل من الواقع بـ5.8%).
4. ☐ **D3/D4**: `gpt-5.2/5.2-codex/5.3-codex/5.1-codex-mini/5-codex-mini` في مجموعة الرقاقات (patch) بحد أقصى **1536 بدون `original`** (الكود يفترض 10000)؛ `gpt-5-codex-mini` في النظام الخاطئ (بلاطة ← رقاقة).

6. ☐ **دعم Gemini** (جديد): `src/core/gemini.ts` + `gemini-model-profiles.ts` + مسارات `:generateContent`/`:streamGenerateContent` في البروكسي. هندسة موثّقة: **1152×1536 (وحدة قص دقيقة 768، 4 بلاطات، 42.2 حرفًا/رمز — أفضل نسبة موثّقة بين المزوّدين الثلاثة)**؛ رهانات للمعايرة: 768² مع `media_resolution:MEDIUM` (56.4) وGemini 3 HIGH. تنبيه: نقطة نهاية Gemini المتوافقة مع OpenAI ستمر عبر محوّل OpenAI بفوترة خاطئة.

- ☐ **TS في كل مكان**: النواة بالفعل TS، تحويل `eval/*.mjs`، `demo/`، `scripts/*.mjs`، `bench/` (النمط: tsx + vitest؛ `benchmarks/density-frontier/` وُلدت هكذا).

- ☐ **صور GIF بدل الفيديوهات** في الـ README (تسجيل بـ vhs/asciinema+agg؛ مقارنة جنبًا إلى جنب عادي مقابل البروكسي).
- ☐ لوحة تحكم v2 (إعادة تنفيذ عبر HTTP API — دون وراثة كود طرف ثالث): مُطلِق "افتح طرفية بـ ANTHROPIC_BASE_URL"، فحص "هل تمر الحركة عبري؟"، فاحص صورة-مقابل-نص، جلسات، لوحة تكلفة بالعملة، i18n خفيف، SSE بدل الاستطلاع (polling)، استمرارية SQLite مع احتفاظ (مخططها ذو 24 عمودًا نقطة انطلاق جيدة).
- ☐ أفكار مصغّرة من dense-image-gen: وضع `lines` (تخطيط محفوظ للكود/الجداول)، `--keep-ws`، عنوان أصل لكل صفحة ("system prompt" / "tool docs" / "history turn N")، CLI مستقل `render arquivo.md -o out.png`.

## المرحلة 5 — النقل إلى OmniRoute

- ☐ محرك `CompressionEngine` (قالب `cavemanAdapter.ts`)، مسجَّل في `engines/index.ts` + `engineCatalog.ts`؛ `targets: ["messages","tool_results"]`، `applyAsync`.
- ☐ التوصيل: تمرير `supportsVision` في `chatCore.ts:1297` (سطر واحد) أو حلّه عبر `isVisionModelId`.
- ☐ ترتيب التكديس: الأخير (محركات RTK/Caveman/الدلالية أولًا؛ OmniGlyph يحوّل المتبقي إلى صور).
- ☐ الثوابت: لا تُعِد كتابة الكتل التي تحمل `cache_control` الخاص بالعميل أبدًا (درس #4560)؛ بوّابة الدقة (#5127) تحتاج إعفاءً معلَنًا أو ورقة حقائق نصية تحقّق الثوابت؛ قياس عن بُعد للمحاولات مع `skip_reason` (درس #4268).
- ☐ التوجيه: التراجع/إعادة المحاولة بعد المحرك يجب أن يحترم قدرة الرؤية والقائمة المسموحة (إعادة الضغط أو التجاوز).
- ☐ تآزر CCR: `emitRecoverable` ← مخزن CCR مع استرجاع لكل شريحة (`head/tail/grep`، #5187) = إعادة توسيع انتقائية كاملة.
- ☐ توسيع الحصة المجانية كميزة تسويقية: كل رمز من الحصة المجانية يمنح ~2-3× أحرفًا أكثر على نماذج الرؤية؛ حصة Gemini المجانية + هندسة 1152×1536 هي أقوى حالة.

النسخ الحرفي على الصفحة القياسية — 100% مُصفّى — ولا
يُفعَّل على الصفحة الكبيرة؛ الصياغة مهمة. (2) الامتناع يعمل: 20×
ILEGIVEL مقابل 5 اختلاقات على الصفحة الكبيرة. (3) Opus عند 10×16 يقرأ 78%
بدقة (n=9) مقابل 0% تاريخيًا عند 5×8 — أول دليل مباشر على نقطة الانحناء (knee).
(4) عدم وضوح الصفحة الكبيرة عبر OpenRouter يوحي بأن النقل يعيد
أخذ العينات (RESAMPLE) (Bedrock/Vertex المستوى القياسي؟) — فرضية حاسمة يجب اختبارها على
API المباشر لـ Anthropic؛ هندسة A/B تبقى مفتوحة حتى ذلك الحين. نفدت حصة
OpenRouter منتصف ذراع Opus.
