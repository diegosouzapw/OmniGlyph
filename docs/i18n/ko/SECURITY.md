## 위협 모델 (OmniGlyph란 무엇인가)

OmniGlyph는 사용자의 클라이언트(예: Claude Code)와 LLM API 사이의
**로컬 프록시**입니다. 설계상 세션의 모든 콘텐츠와 전송 중인 자격 증명을
볼 수 있습니다. 이에 따른 보안 결정 사항:

- **기본적으로 루프백에 바인딩**(`127.0.0.1`): 대시보드에는 인증이 없으며
  캡처된 세션 컨텍스트(이미지의 소스 텍스트, 텔레메트리)를 제공합니다.
  `HOST=0.0.0.0`은 명시적인 옵트인이며 이 모든 것을 네트워크에
  노출시킵니다 — 신뢰할 수 있는 네트워크에서만 사용하세요.
- **자격 증명**: 프록시는 클라이언트의 인증 헤더를 업스트림으로 전달하며
  이를 영속화하지 않습니다. 환경 변수로 제공된 키(`ANTHROPIC_API_KEY` 등)는
  메모리에만 남습니다.
- **로컬 텔레메트리**: `~/.omniglyph/events.jsonl`은 요청별 메타데이터
  (토큰 수, 바디 해시)와, 4xx 오류 시 압축된 바디 샘플을 보관합니다 — 이
  파일을 민감한 정보로 취급하세요.
- **이미지화된 콘텐츠는 손실이 있습니다**: 바이트 단위로 정확해야 하는 값
  (시크릿, 해시)은 절대 이미지 판독에 의존해서는 안 됩니다. 파이프라인은
  이를 텍스트로 유지하지만, 황금 규칙은 다음과 같습니다: LLM 컨텍스트에
  시크릿을 넣지 마세요.
- **공급망**: `pnpm-workspace.yaml`은 새로 추가되는 모든 패키지에 대해
  `minimumReleaseAge` 3일을 강제합니다. core는 런타임 의존성이 단 하나뿐
  입니다.

## 지원되는 버전

최신 릴리스 라인(`main` / 가장 최근의 `v1.x`)만 수정 사항을 받습니다.
