# OmniGlyph — 통합 측정 결과 (2026-07-05)

🌐 번역: [모든 언어](../../../README.md)

이 세션에서 측정된 모든 내용을 출처와 n값과 함께 정리했으며, 가설은 맨 끝에
명확히 구분되어 있습니다. 근거 자료: `benchmarks/billing-sweep/results/`와
`benchmarks/density-frontier/results/`(답변별 JSONL).

## TL;DR — 두 개의 막대로 보는 전체 결과

**비용** — 표준 1568×728 페이지 한 장은 28,080자를 담아 고정 1,460
토큰이 들며, 같은 텍스트를 원문 그대로 보내면 ~10배 더 비쌉니다:

```
same 28,080-char context

  as dense TEXT   ██████████████████████████████████████████████  ~14,040 tokens
  as ONE IMAGE    █████                                              1,460 tokens   (flat, WYSIWYG)
```

**정확도** — 하지만 모델이 실제로 페이지를 읽을 때만 그렇습니다.
게이트는 fail-closed 방식이며, ✅ 행만 출시됩니다:

```
  Fable 5 · 1-bit std page (prod)  ██████████████████████████████  30/30  ✅
  Fable 5 · AA std page (old)      █████████████████████████░░░░░  25/30  🟡 5 abstain
  Opus 4.8 · 10×16 (safe mode)     ████████████████████████░░░░░░  ~24/30 ⚠️
  Fable 5 · high-res 1928²         █░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  ~2/30  🚫 billing trap
  GPT-5.5 / Gemini 2.5-flash       ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  0      ⛔ blocked
```

이 문서의 나머지 부분은 이 두 막대 뒤에 있는 근거 자료입니다.

## 1. Anthropic 과금 (직접 count_tokens, $0, 기하 구조 11개 × 모델 2개)

확인된 공식: 티어별 리사이즈 이후 `tokens = ceil(w/28) × ceil(h/28)`,
**Fable 5는 블록당 +3 / Sonnet 4.5는 블록당 +4** — 모든 행에서 잔차 0.

|---|---|---:|---:|

| 문서 앵커 | 1000×1000 | 1299 | 1300 |
| 표준 페이지 | 1568×728 | 1459 | 1460 |
| **큰 페이지** | **1928×1928** | **4764 (WYSIWYG!)** | 1525 (리샘플) |

## 2. 판독 정확도 (density-frontier, hex/camelCase/숫자 니들 + 방해 요소)

### Fable 5 2×2 매트릭스 — CLI/구독 경유, arm당 n=30, 동일 코퍼스(~16.6k자)

| 페이지 × 아틀라스 | 정확 | 기권(ILEGIVEL) | 은밀한 오류 |
|---|---:|---:|---:|
| 표준 1568×728 · **1비트** | **30/30 (100%)** | 0 | 0 |
| 표준 1568×728 · AA | 25/30 (83%) | 5 | 0 |
| 고해상도 1928×1928 · **1비트** | 20/30 (67%) | 10 | 0 |
| 고해상도 1928×1928 · AA | 0/30 | 29 | 1 (매트릭스에 의해 예측됨) |

→ **두 페이지 모두에서 1비트 > AA; 질문 120개 전체에서 컨퓨불레이션 0건.**
적용됨: `DENSE_RENDER_STYLE` → `aa:false`(커밋 9a25585).
⚠️ 고해상도는 전송 계층의 리샘플로 인해 저하된 상태로 도달합니다(H1/H3
참고); 67%는 상한이 아니라 하한입니다.

### Opus 4.8 — CLI/구독 경유, arm당 n=30

| 설정 | 정확 | 기권 | 오류 |
|---|---:|---:|---:|
| 고해상도 · 10×16 셀 | **26/30 (87%)** | 0 | 4 (숫자) |
| 표준 · 5×8 셀 | 0/30 | 30 | 0 |

→ 자체 n으로 Opus의 무릎(knee) 지점 확인(업스트림은 n=20에서 10×16 기준
95% 측정). "Opus 세이프 모드"는 실행 가능함: 하네스 코퍼스에서 큰 페이지의
10×16은 이미지 토큰당 약 1.7자.

### OpenRouter 경유 (동일 코퍼스/질문) — 가독성에 대해서는 결론 없음

| 측정된 사실 | 숫자 |
