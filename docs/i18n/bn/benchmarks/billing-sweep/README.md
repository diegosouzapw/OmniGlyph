# Anthropic ভিশন-বিলিং সুইপ

দুটি খোলা জ্যামিতি প্রশ্নের সিদ্ধান্ত নেয় এমন ফ্রি `count_tokens` সুইপ:

1. **ফর্মুলা** — API কি `ceil(w/28) × ceil(h/28)` প্যাচ বিল করে (বর্তমান
   ডক) নাকি অবসৃত `w·h/750`? প্রোব সেট প্রতি সারিতে 25–180 টোকেন দ্বারা
   দুটিকে আলাদা করে।
2. **টিয়ার** — `claude-fable-5` কি হাই-রেজোলিউশন ক্যাপ পায় (লং এজ
   ≤ 2576 px, ≤ 4784 ভিজুয়াল টোকেন)? `page-old-1928x1928` সারিটি নির্ণায়ক:
   ≈ **4761** মাপা মানে হাই-রেজ WYSIWYG (পুরনো বড় পৃষ্ঠা আজকের 1568×728-এর
   চেয়ে প্রতি ইমেজে ~3.3× বেশি chars ধরে, একই chars/token-এ); ≈ **1521**
   মানে স্ট্যান্ডার্ড-টিয়ার resample, এবং 1568×728 সঠিক থেকে যায়।

প্রেক্ষাপট: বর্তমান 1568×728 পৃষ্ঠার পেছনে থাকা 2026-07-01 সুইপ
(পাঠযোগ্যতা অডিট, 2026-07-01) `claude-sonnet-4-5`-এ মাপা হয়েছিল — একটি
স্ট্যান্ডার্ড-টিয়ার মডেল — যখন প্রোডাকশন লক্ষ্য করে Fable 5-কে, যাকে ভিশন
ডক হাই-রেজোলিউশন টিয়ারে রাখে। সেই অডিটও বর্তমান পৃষ্ঠাকে 1460 টোকেনে
মাপা হয়েছিল: /750-এর 1522-এর চেয়ে প্যাচ ফর্মুলার 1456-এর কাছাকাছি,
ইঙ্গিত দেয় API ইতিমধ্যে প্যাচ বিলিংয়ে চলে গিয়েছিল।

## চালান

```bash
pnpm run build                              # dist/ prerequisite (like all evals)
node benchmarks/billing-sweep/run.mjs --dry-run   # predictions only, no key, $0

ANTHROPIC_API_KEY=sk-... node benchmarks/billing-sweep/run.mjs \
  --models claude-fable-5,claude-sonnet-4-5 --probe-multi --probe-20plus
```

**সরাসরি** API-তে হিট করতে হবে — কখনো OmniGlyph প্রক্সির মাধ্যমে নয়, যা
বডি রূপান্তর করে দেবে। `count_tokens` ফ্রি; সম্পূর্ণ সুইপ ~25টি রিকোয়েস্ট
করে।

## আউটপুট পড়া

প্রতি মডেলে, প্রতিটি প্রোব সারি চারটি প্রেডিকশনের (`patch`/`legacy750` ×
`standard`/`highres`) বিপরীতে মাপা ইমেজ টোকেন (ইমেজ-সহ বিয়োগ টেক্সট-শুধু
বেসলাইন) দেখায়; সারাংশ গড় absolute residual অনুযায়ী হাইপোথিসিস র‍্যাঙ্ক
করে। `--probe-multi` প্রতি-ইমেজ ক্যাপ চেক করে (2×1092² ≈ 2×1521);
`--probe-20plus` >20-ইমেজ নিয়ম চেক করে (একটি >2000 px পার্শ্ব প্রত্যাখ্যাত
হতে হবে, resampled নয়)। সারিগুলো `results/*.jsonl`-এ যায়; প্রেডিকশন গণিত
`formulas.mjs`-এ থাকে, `tests/billing-sweep-formulas.test.ts` দ্বারা
পিনড।
