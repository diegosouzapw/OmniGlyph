# OmniGlyph-এ অবদান রাখা

আপনার আগ্রহের জন্য ধন্যবাদ! এই প্রজেক্টের দুটি অ-আলোচনাযোগ্য কালচার নিয়ম আছে —
সেগুলোই কারণ যে README-এর প্রতিটি সংখ্যা বিশ্বাসযোগ্য।

## নিয়ম ১ — কঠোর TDD

সব প্রোডাকশন কোড প্রথমে ব্যর্থ হওয়া একটি টেস্ট থেকে জন্ম নেয়:

1. টেস্ট লিখুন এবং **সঠিক কারণে এটি ব্যর্থ হতে দেখুন**।
2. এটি পাস করানোর জন্য সর্বনিম্ন কোড লিখুন।
3. সবুজ থাকা অবস্থায় রিফ্যাক্টর করুন।

পূর্ণ মানদণ্ড হলো: `pnpm run typecheck && pnpm test && pnpm run build` — সবসময়
এই তিনটিই (docs link-lint এবং rebrand গার্ড `tests/docs-integrity.test.ts`-এর
মাধ্যমে `pnpm test`-এর ভেতরে চলে)।

## নিয়ম ২ — দাবির আগে পরিমাপ

জ্যামিতি, অ্যাটলাস, বিলিং ফর্মুলা, বা মডেল স্কোপে কোনো পরিবর্তন একটি পরিমাপকৃত
সংখ্যা ছাড়া ল্যান্ড করে না। রিপোজিটরি এই শৃঙ্খলার চারপাশে তৈরি:

- বিলিং খরচ → `benchmarks/billing-sweep/`-এর মাধ্যমে প্রমাণ করুন (`count_tokens`
  ফ্রি; প্রত্যাশিত অবশিষ্টাংশ: শূন্য)।
- পাঠযোগ্যতা → `benchmarks/density-frontier/`-এর মাধ্যমে প্রমাণ করুন (প্রতি আর্মে
  n≥30, ডিটারমিনিস্টিক স্কোরিং, `benchmarks/*/results/`-এ কমিটেড JSONL রসিদ)।
- প্রোডাকশন ডিফল্ট পরিবর্তনের গ্রহণযোগ্যতার মানদণ্ড: gist == টেক্সট বেসলাইন
  **এবং** শূন্য নীরব এক্সাক্ট-স্ট্রিং এরর **এবং** ধনাত্মক সাশ্রয়।

সংখ্যাবিহীন হাইপোথিসিস `docs/ROADMAP.md`-এ হাইপোথিসিস হিসেবে যায় — কখনো
README-এ সত্য হিসেবে নয়। দুটি "সুস্পষ্ট" ধারণা ইতিমধ্যে ডেটা দিয়ে খণ্ডন করা
হয়েছে (হাই-রেজ পৃষ্ঠা, অ্যান্টি-অ্যালিয়াসড অ্যাটলাস); প্রক্রিয়াটি কাজ করে।

## সেটআপ

```bash
pnpm install
pnpm test              # full suite, ~40–90s
pnpm run dev:node      # local proxy in watch mode
```

Node ≥18 (CI 20/22/24 টেস্ট করে), pnpm 10 (package.json-এর `packageManager`
দ্বারা পিন করা)।

## গঠন

| ফোল্ডার | নিয়ম |
|---|---|
| `src/core/` | রানটাইম-অ্যাগনস্টিক (শুধু Web API — Node এবং Workers-এ চলে) |
| `src/node.ts` / `src/worker.ts` | শুধুমাত্র হোস্ট প্লাম্বিং |
| `benchmarks/` | পুনরায় চালানো যায় এমন হার্নেস; JSONL ফলাফল রসিদ, কমিটেড |
| `docs/` | benchmarks/ (সংখ্যা), architecture/ (মানচিত্র), ROADMAP (হাইপোথিসিস), ops/ (OmniRoute) |

## কমিট ও PR

- কনভেনশনাল কমিট (`feat:`, `fix:`, `perf:`, `docs:`, `refactor:`,
  `test:`, `chore:`), বডিতে প্রাসঙ্গিক সংখ্যাসহ *কেন*-এর ব্যাখ্যা।
- ছোট, ফোকাসড PR; আচরণ পরিবর্তন সেই টেস্টের সাথে আসে যা এটিকে পিন করে এবং,
  প্রযোজ্য ক্ষেত্রে, সেই বেঞ্চমার্ক যা এটিকে যুক্তিসঙ্গত করে।
- ক্লায়েন্টের `cache_control` ব্লক পুনর্লিখন করবেন না, আলোচনা ছাড়া রানটাইম
  ডিপেন্ডেন্সি যোগ করবেন না (কোরটি ইচ্ছাকৃতভাবে ডিপেন্ডেন্সি-লাইট), রেন্ডার
  পাথে `Math.random`/টাইমস্ট্যাম্প ব্যবহার করবেন না (ডিটারমিনিজম একটি কঠোর
  ইনভেরিয়েন্ট, বাইট-আইডেন্টিটি দ্বারা টেস্ট করা)।
