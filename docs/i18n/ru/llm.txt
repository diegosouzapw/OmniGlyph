# OmniGlyph

> Прокси сжатия контекста в изображение для LLM. Локальный прокси, который
> переписывает объёмные части каждого запроса (системный промпт, документацию
> инструментов, старую историю, крупные результаты вызовов инструментов) в
> плотные 1-битные PNG-страницы до того, как они покинут машину, используя
> точную формулу биллинга по каждому провайдеру (Anthropic/OpenAI/Gemini), так
> что конвертация происходит только тогда, когда математика токенов выигрывает.
> Измеренное итоговое снижение счёта: 59–70%. Часть семейства OmniRoute;
> также поставляется как движок сжатия `omniglyph` внутри OmniRoute.

**Ключевая ценность:** направьте любой Anthropic-совместимый инструмент на
`ANTHROPIC_BASE_URL=http://127.0.0.1:47821`, и объёмный контекст будет
тарифицироваться как токены изображения (~19 символов/токен) вместо токенов
текста (~2 символа/токен).

**Текущая версия:** 1.0.0 · **Лицензия:** MIT · **Среда выполнения:** Node ≥18
и Cloudflare Workers.

## Как это работает

1. Гейт рентабельности: вычисляет точный биллинг изображения провайдера
   (Anthropic: ⌈w/28⌉ × ⌈h/28⌉ + оверхед блока, измерено с нулевой невязкой)
   и конвертирует только тогда, когда это экономит токены.
2. Гейт моделей (fail-closed): изображения получают только модели, прошедшие
   бенчмарк чтения. Одобрены сегодня: Claude Fable 5 (100% точных чтений).
   Заблокированы с доказательствами: GPT-5.5 (0/60), Gemini 2.5-flash
   (конфабулирует), Opus 4.8 только в безопасном режиме по опции (4× глифы).
3. Рендер: reflow + 1-битный атлас глифов 5×8 → стандартные PNG-страницы
   1568×728 (тир высокого разрешения — измеренная биллинговая ловушка, не
   используется).
4. Сплайсинг страниц обратно дружелюбным к кэшу образом; побайтово точные
   значения (хеши/id) едут рядом как текст; неудачные чтения абстинируются
   (ноль незаметных конфабуляций из ~300 проб).
5. Повторная попытка при отказе: если модель отказывается от отрендеренной
   страницы, исходный запрос автоматически повторяется.

## Структура проекта

- `src/core/` — конвейер трансформации, точная формула биллинга, рендерер,
  LRU-кэш рендера, прокси с повторной попыткой при отказе. Формула биллинга
  экспортируется в корне пакета.
- `src/node.ts` — локальный хост с дашбордом (сэкономленные токены,
  конвертации бок о бок, аварийный выключатель). `src/worker.ts` — хост для
  Cloudflare Workers.
- `benchmarks/` — наборы billing-sweep и density-frontier; за каждой
  опубликованной цифрой стоит JSONL-подтверждение в `benchmarks/*/results/`.
- `docs/` — architecture/, benchmarks/ (методология + результаты), ops/
  (руководство по OmniRoute), i18n/pt-BR/.

## Важные заметки для LLM

- Строгий TDD и измерение прежде утверждений: цифры без бенчмарк-подтверждений
  не принимаются; fail-closed гейты никогда не должны ослабляться ради
  прохождения тестов.
- Сжатие происходит только на стороне запроса — ответы модели проходят через
  прокси нетронутыми.
- Прокси локален прежде всего: никакой дополнительной информации не
  отправляется какой-либо третьей стороне.
