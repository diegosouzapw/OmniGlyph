## TL;DR — весь результат в двух полосках

**Стоимость** — одна стандартная страница 1568×728 несёт 28,080 символов за
фиксированные 1,460 токенов; тот же текст, отправленный сырым, стоит ~в 10×
дороже:

```
same 28,080-char context

  as dense TEXT   ██████████████████████████████████████████████  ~14,040 tokens
  as ONE IMAGE    █████                                              1,460 tokens   (flat, WYSIWYG)
```

**Точность** — но только там, где модель действительно читает страницу.
Гейт fail-closed; отгружается только строка ✅:

```
  Fable 5 · 1-bit std page (prod)  ██████████████████████████████  30/30  ✅
  Fable 5 · AA std page (old)      █████████████████████████░░░░░  25/30  🟡 5 abstain
  Opus 4.8 · 10×16 (safe mode)     ████████████████████████░░░░░░  ~24/30 ⚠️
  Fable 5 · high-res 1928²         █░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  ~2/30  🚫 billing trap
  GPT-5.5 / Gemini 2.5-flash       ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  0      ⛔ blocked
```

Остальная часть этого документа — подтверждения, стоящие за этими двумя
полосками.

## 1. Биллинг Anthropic (прямой count_tokens, $0, 11 геометрий × 2 модели)

Подтверждённая формула: `tokens = ceil(w/28) × ceil(h/28)` после ресайза по
тиру, **+3/блок (Fable 5) / +4/блок (Sonnet 4.5)** — НУЛЕВАЯ невязка по всем
строкам.

| проба | размеры | Fable 5 (high-res) | Sonnet 4.5 (standard) |
|---|---|---:|---:|
| doc anchor | 1092×1092 | 1524 | 1525 |
| doc anchor | 1000×1000 | 1299 | 1300 |
| standard page | 1568×728 | 1459 | 1460 |
| **large page** | **1928×1928** | **4764 (WYSIWYG!)** | 1525 (resample) |
| hi-res ceiling | 1960×1960 | 4764 (clamp) | 1525 |
| hi-res long edge | 2576×1204 | 3959 | 1516 |
| tall strip | 768×1932 | 1935 | 1292 (resample) |
| 2×1092² (multi) | — | 3048 (= 2×1524) | 3049 |
| 21×280² (>20 imgs) | — | 2163 | 2164 |
| 20×280²+2100² | — | 6824 (2100²→downscale, НЕ отклонено в count_tokens) | 3585 |

Производные решения (реализованные): точный per-patch гейт; тир по модели
(Fable/Mythos/Opus 4.8/4.7/Sonnet 5 = high-res); `cols` 313→312.

## 2. Точность чтения (density-frontier, needles hex/camelCase/цифры + отвлекающие факторы)

### Матрица 2×2 Fable 5 — через CLI/подписку, n=30/плечо, тот же корпус (~16.6k символов)

| страница × атлас | точно | абстенции (ILEGIVEL) | незаметные ошибки |
|---|---:|---:|---:|
| standard 1568×728 · **1-bit** | **30/30 (100%)** | 0 | 0 |
| standard 1568×728 · AA | 25/30 (83%) | 5 | 0 |
| high-res 1928×1928 · **1-bit** | 20/30 (67%) | 10 | 0 |
| high-res 1928×1928 · AA | 0/30 | 29 | 1 (предсказано матрицей) |

→ **1-bit > AA на обеих страницах; ноль конфабуляций на 120 вопросах.**
ПРИМЕНЕНО: `DENSE_RENDER_STYLE` → `aa:false` (коммит 9a25585).
⚠️ high-res приходит деградированным из-за transport resample (см. H1/H3) —
67% — это пол, а не потолок.

### Opus 4.8 — через CLI/подписку, n=30/плечо

| конфигурация | точно | абстенции | ошибки |
|---|---:|---:|---:|
| high-res · ячейка 10×16 | **26/30 (87%)** | 0 | 4 (цифры) |
| standard · ячейка 5×8 | 0/30 | 30 | 0 |

→ «Колено» Opus подтверждено нашим собственным n (upstream измерил 95% при
10×16 с n=20). «Безопасный режим Opus» жизнеспособен: 10×16 на большой
странице ≈ 1.7 символа на токен изображения на корпусе набора.

### Через OpenRouter (тот же корпус/вопросы) — неокончательно для читаемости

| измеренный факт | число |
|---|---|
| content_filter на вопросах транскрипции (стандартные страницы) | 60/60 (100%) |
| content_filter на high-res страницах | 5-6/30 (~20%) |
| Fable high-res: абстенции + ошибки | 20 ILEGIVEL + 5 ошибок (2 предсказано) |
| Opus 10×16 (до окончания кредитов) | 7/9 точно (78%) |
| Ошибки чтения, предсказанные матрицей confusability | 4→a, 0→8, S/s регистр |

### Сравнение транспортов (тот же вопрос, тот же контент)

| транспорт | фильтр/отказ | большая страница читаема? |
|---|---|---|
| Прямой API (n=9, до окончания кредитов) | 0 | не тестировано |
| OpenRouter | ~100% std / ~20% hi-res | нет (подозревается: resample) |
| Claude Code CLI (подписка) | 0 content_filter; ~50% крупных батчей зависали (решено чанками по 10 + retry) | нет (подозревается: ресайз Read) |

токены промпта.

## ДОПОЛНЕНИЕ 2026-07-06 (3) — Gemini 2.5-flash (ЧАСТИЧНО: бесплатная квота истощилась в середине прогона)

Из ~26 ответов-изображений, которые прошли до истощения квоты: **0
правильных, 1 абстенция, ~25 КОНФАБУЛЯЦИЙ** — и это не спутанные глифы: это
случайные цифры (`indexLedgerInd → 0040375615`), т.е. энкодер видит почти
ничего на тестируемых плотностях (native tile 42 символа/токен и MEDIUM
flat), а 2.5-flash ВЫДУМЫВАЕТ вместо того, чтобы воздержаться (игнорирует
инструкцию ILEGIVEL). Текстовый контроль: 3/3 на тех, что прошли. Без
инфляции вывода (6-28 токенов/ответ).

Предварительный сигнал: H5/H6 склоняются к НЕТ на 2.5-flash, с режимом
отказа ХУЖЕ, чем у GPT (незаметная конфабуляция вместо абстенции) — Gemini
потребует дополнительных защитных мер в прокси. В ожидании закрытия:
перезапустить с платной квотой или в другой день, а также протестировать
gemini-2.5-pro (flash — самый слабый читатель в семействе). Страница
native-tile всё ещё имеет лучшее ЗАДОКУМЕНТИРОВАННОЕ соотношение (42.3
символа/токен); под вопросом именно читаемость.

Заметка о стоимости: неполные страницы (последняя в корпусе) тарифицируются
плохо в режиме tile (малая высота → малая единица кропа → больше тайлов) —
паддинг последней страницы до высоты 1152px — обязательная оптимизация,
если Gemini войдёт в проект.
