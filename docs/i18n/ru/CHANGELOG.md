# Журнал изменений

Формат: [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) · семантическое версионирование.

## [1.0.0] — 2026-07-07

Первый публичный релиз.

### Продукт

- **Прокси сжатия контекста в изображение**: переписывает объёмные части
  каждого LLM-запроса (системный промпт, документацию инструментов, старую
  историю, крупные результаты вызовов инструментов) в плотные 1-битные
  PNG-страницы до того, как они покинут вашу машину. Локальный сервер на Node
  и хост для Cloudflare Workers.
- **Точная формула биллинга по каждому провайдеру** (`src/core/`): у Anthropic
  патчи 28px + оверхед 3–4 токена/блок (собственный sweep, нулевая невязка),
  формулы OpenAI и Gemini проверены по официальной документации. Экспортируется
  в корне пакета (`anthropicImageTokens`, `resolveAnthropicVisionTier`, лимиты
  тиров).
- **Измеренная продакшн-конфигурация рендера**: плотный 1-битный атлас глифов
  (без сглаживания), страницы стандартного тира — каждый выбор подкреплён
  бенчмарк-подтверждением в `benchmarks/*/results/`.
- **Бенчмарк-наборы** (`benchmarks/`): billing-sweep (учёт токенов) и
  density-frontier (граница точности чтения по моделям/плотностям),
  перезапускаемые через API, OpenRouter, Claude Code CLI или через OmniRoute
  (`--via-omniroute`).
- **Повторная попытка при отказе**: SSE/JSON-снифер повторяет исходный запрос,
  когда модель отказывается читать отрендеренную страницу (аварийный
  выключатель `retryRefusalWithOriginal`).
- **LRU-кэш рендера** для детерминированных страниц.
- **Движок OmniRoute**: поставляется как движок сжатия `omniglyph` в
  [OmniRoute](https://github.com/diegosouzapw/OmniRoute) (одиночный режим и
  стек конвейера), с fail-closed гейтами и учётом токенов с поддержкой
  изображений.

### Цифры (всё воспроизводимо)

- Пример рендера UI: 1015 символов → PNG 438×120, 254 → 84 токена (**экономия
  66.9%**).
- Стандартная страница 1568×728 = 1456 токенов изображения независимо от того,
  сколько текста она содержит.
- Claude читает плотные 1-битные страницы со 100% точностью на продакшн-
  плотности; Opus 4.8 читает 77–87% при 10×16.

### Отрицательные решения (измеренные, а не мнения)

- **Тир высокого разрешения — биллинговая ловушка**: страница 1928² тарифицируется
  WYSIWYG, но энкодер не получает полное разрешение — оба тира рендерят
  стандартные страницы.
- **GPT-5.5 отклонён**: 0/60 чтений плотной полосы и раздувание завершения
  примерно в 40× по сравнению с текстовым контролем.
- **gpt-4o-mini никогда не переводится в изображение** (порог 2833/5667
  токенов делает это нерентабельным).
- **Gemini 2.5-flash конфабулирует** вместо того, чтобы воздержаться на
  плотных страницах (0/26) — ожидает повторного теста на платной квоте.
