## Модель угроз (чем является OmniGlyph)

OmniGlyph — это **локальный прокси** между вашим клиентом (например, Claude
Code) и API LLM. По конструкции он видит весь контент вашей сессии и ваши
учётные данные при передаче. Соответствующие решения по безопасности:

- **По умолчанию привязан к loopback** (`127.0.0.1`): у дашборда нет
  аутентификации, и он раздаёт захваченный контекст сессии (исходный текст
  изображений, телеметрию). `HOST=0.0.0.0` — это явный опциональный выбор, и
  он открывает всё это сети — используйте только в доверенной сети.
- **Учётные данные**: прокси пересылает заголовки авторизации клиента
  вышестоящему сервису и не сохраняет их. Ключи, переданные через переменные
  окружения (`ANTHROPIC_API_KEY` и т. п.), остаются в памяти.
- **Локальная телеметрия**: `~/.omniglyph/events.jsonl` хранит метаданные по
  каждому запросу (количество токенов, хеши тела) и, при ошибках 4xx, сжатые
  образцы тела — относитесь к этому файлу как к чувствительному.
- **Контент в виде изображения — lossy**: побайтово точные значения (секреты,
  хеши) никогда не должны зависеть от чтения изображения; конвейер сохраняет
  их как текст, но золотое правило: не помещайте секреты в контекст LLM.
- **Цепочка поставок**: `pnpm-workspace.yaml` обеспечивает `minimumReleaseAge`
  в 3 дня для любого нового пакета; у ядра единственная зависимость времени
  выполнения.

## Поддерживаемые версии

Только последняя линия релиза (`main` / самая свежая `v1.x`) получает
исправления.
