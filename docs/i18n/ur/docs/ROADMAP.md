- ☑ `benchmarks/density-frontier/` ہارنس (آف لائن لاگت × درستگی API کے ذریعے، الجھن پیدا کرنے والے مشتتات کے ساتھ سوئیاں، متعین اسکورنگ)۔

## مرحلہ 1 — کثیر-فراہم کنندہ بلنگ فکسز (آڈٹ میں تصدیق شدہ بگز)

آڈٹ (سرکاری دستاویزات 2026-07-05 کو حاصل کی گئیں) کے ذریعے مقرر کردہ ترجیح:

1. ☐ **D2 (الٹا گیٹ)**: `gpt-4o-mini` ڈیفالٹ ٹائل 85/170 میں گرتا ہے، لیکن اس کی لاگت **2833 بیس / 5667 فی ٹائل** ہے (~33× کم اندازہ، ~0.8 حرف/ٹوکن) — اس پر امیجنگ ہمیشہ نقصان دہ ہے اور گیٹ اسے منظور کرتا ہے۔ `src/core/gpt-model-profiles.ts:51-59`۔
2. ☐ **D5**: `detail:'original'` غیر مشروط طور پر بھیجا جاتا ہے (`src/core/openai.ts:392,402`)، لیکن یہ صرف gpt-5.4+ میں موجود ہے؛ اسے پروفائل سے اخذ کریں۔
3. ☐ **D1**: `o4-mini` ملٹی پلائر 1.62 → **1.72** (5.8% کم اندازہ)۔
4. ☐ **D3/D4**: `gpt-5.2/5.2-codex/5.3-codex/5.1-codex-mini/5-codex-mini` پیچ بکٹ میں ہیں **کیپ 1536 بغیر `original` کے** (کوڈ 10000 فرض کرتا ہے)؛ `gpt-5-codex-mini` غلط ریگیم میں ہے (ٹائل → پیچ)۔
5. ☐ **GPT جیومیٹری**: `GPT_MAX_HEIGHT_PX 1932 → 2048` (دونوں ریگیمز کے ساتھ ہم آہنگ: 64×32 پیچز اور 4×512 ٹائلز؛ +6.25% مفت حروف)۔ مخصوص 5.4/5.5 `original` پروفائل: 1568×5984 تک (9,163 پیچز ≤ 10k، ~233k حروف ایک بلاک میں) — پہلے پڑھنے کی صلاحیت کا A/B۔
6. ☐ **Gemini سپورٹ** (نیا): `src/core/gemini.ts` + `gemini-model-profiles.ts` + پراکسی میں `:generateContent`/`:streamGenerateContent` روٹس۔ دستاویزی جیومیٹری: **1152×1536 (عین کراپ یونٹ 768، 4 ٹائلز، 42.2 حروف/ٹوکن — 3 فراہم کنندگان میں بہترین دستاویزی تناسب)**؛ کیلیبریٹ کرنے کے لیے شرطیں: `media_resolution:MEDIUM` (56.4) کے ساتھ 768² اور Gemini 3 HIGH۔ احتیاط: Gemini کا OpenAI-موافق اینڈ پوائنٹ غلط بلنگ کے ساتھ OpenAI ٹرانسفارمر سے گزرے گا۔

## مرحلہ 2 — پڑھنے کا معیار (جج کے طور پر density-frontier ہارنس)

- ◐ Fable پر فیصلہ کن std بمقابلہ high-res A/B (چل رہا ہے؛ معیار: gist == متن اور صفر خاموش-غلط اور بچت > 0)۔
- ☐ گھنے راستے میں AA بمقابلہ 1-بٹ تضاد حل کریں (کوڈ کہتا ہے "eval-only"، پروڈکشن AA استعمال کرتا ہے)۔
- ☐ (وجہ کے ساتھ 2026-07-06 کو ملتوی) گلف سرجری: پروڈکشن کنفیگ 30/30 پڑھتا ہے — آج سرجری کے حل کرنے کے لیے کوئی قابلِ پیمائش خرابی نہیں ہے۔ اگر sub-100% ہدف دائرہ کار میں آئے (مثلاً Opus) یا نئی پیمائشیں پسپائی دکھائیں تو دوبارہ جائزہ لیں۔
- ☑ ~~Light-theme A/B~~ معائنے سے حل شدہ (2026-07-06): رینڈر پہلے سے ہی سیاہ-پر-سفید ہے (render.ts:635/822، بلٹ کے بعد اُلٹا) — ادب کے ساتھ ہم آہنگ؛ مفروضہ ایک غلط بنیاد (اپ اسٹریم مثال کی تصویر) سے پیدا ہوا تھا۔
- ☐ بائٹ-عین IDs کے لیے چیک سم کے ساتھ ورڈ لسٹ (اپ اسٹریم #38، توثیق شدہ) + گریز کا بینر (#31/#32) + فیکٹ شیٹ میں camelCase (#33/#34)۔
- ☑ پورٹ #45: $schema/$id محفوظ، ہر عنصر کے لیے ٹپلز چھینے گئے (main پر کمٹ)۔
- ☑ انکار-پر-دوبارہ-کوشش (#37/H11): بغیر نقصان کے ری پلے سنیفر + اصل باڈی کے ساتھ ایک دوبارہ کوشش؛ refusalRetried ٹیلی میٹری (main پر کمٹ)۔
- ☐ ری ہائیڈریٹ ٹول (`RecoverableBlock` → کال کرنے کے قابل ٹول؛ LensVLM منتخب دوبارہ توسیع کی توثیق کرتا ہے)۔

- ☐ OmniRoute کوالٹی معیار: eslint 9 + prettier، typecheck/test/build/link-check کے ساتھ CI، CONTRIBUTING، SECURITY، README i18n (پہلے pt-BR)، سیمانٹک CHANGELOG۔

- ☐ ڈیش بورڈ v2 (HTTP API کے ذریعے دوبارہ لاگو کریں — تیسرے فریق کا کوڈ وراثت میں نہ لیں): "ANTHROPIC_BASE_URL کے ساتھ ٹرمینل کھولیں" لانچر، "کیا ٹریفک میرے ذریعے جا رہی ہے؟" چیک، امیج-بمقابلہ-متن انسپیکٹر، سیشنز، کرنسی میں لاگت پینل، ہلکا i18n، polling کی بجائے SSE، رکھ رکھاؤ کے ساتھ SQLite برقراری (اس کا 24-کالم اسکیما ایک اچھا آغاز ہے)۔
- ☐ dense-image-gen سے مائیکرو خیالات: `lines` موڈ (کوڈ/ٹیبلز کے لیے محفوظ لے آؤٹ)، `--keep-ws`، فی صفحہ ماخذ کا عنوان ("system prompt" / "tool docs" / "history turn N")، مستقل CLI `render arquivo.md -o out.png`۔

## مرحلہ 5 — OmniRoute میں پورٹ کریں

- ☐ `CompressionEngine` انجن (`cavemanAdapter.ts` ٹیمپلیٹ)، `engines/index.ts` + `engineCatalog.ts` میں رجسٹرڈ؛ `targets: ["messages","tool_results"]`، `applyAsync`۔
- ☐ پلمبنگ: `chatCore.ts:1297` میں `supportsVision` پاس کریں (1 لائن) یا `isVisionModelId` کے ذریعے حل کریں۔
- ☐ اسٹیک آرڈر: آخری (RTK/Caveman/سیمانٹک رینڈررز پہلے؛ OmniGlyph بقیہ کو امیج کرتا ہے)۔
- ☐ انویریئنٹس: کبھی بھی کلائنٹ کے `cache_control` کے ساتھ بلاکس کو دوبارہ نہ لکھیں (سبق #4560)؛ fidelity گیٹ (#5127) کو ایک اعلانیہ استثنا یا ایک ٹیکسٹ فیکٹ شیٹ درکار ہے جو انویریئنٹس کو مطمئن کرے؛ `skip_reason` کے ساتھ کوشش کی ٹیلی میٹری (سبق #4268)۔
- ☐ روٹنگ: انجن کے بعد fallback/retry کو ویژن صلاحیت اور allowlist کا احترام کرنا چاہیے (دوبارہ کمپریس کریں یا بائی پاس کریں)۔
- ☐ CCR ہم آہنگی: `emitRecoverable` → فی سلائس بازیافت کے ساتھ CCR اسٹور (`head/tail/grep`، #5187) = مکمل منتخب دوبارہ توسیع۔
- ☐ مارکیٹنگ فیچر کے طور پر فری-ٹائر پھیلاؤ: ہر فری-ٹائر ٹوکن ویژن ماڈلز پر ~2-3× زیادہ حروف پیدا کرتا ہے؛ Gemini فری ٹائر + 1152×1536 جیومیٹری سب سے مضبوط کیس ہے۔
