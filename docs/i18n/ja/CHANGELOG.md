## [1.0.0] — 2026-07-07

初回公開リリース。

### プロダクト

- **コンテキストを画像として扱う圧縮プロキシ**: 各LLMリクエストのバルクな部分（system prompt、ツールのドキュメント、古い履歴、大きなツール出力）を、マシンから送信される前に高密度な1ビットPNGページへ書き換えます。ローカルNodeサーバーとCloudflare Workersホストに対応。
- **プロバイダごとの厳密な課金計算**（`src/core/`）: Anthropicは28pxパッチ + ブロックあたり3〜4トークンのオーバーヘッド（自前のスイープで残差ゼロ）、OpenAIとGeminiの式は公式ドキュメントと照合済み。パッケージルートでエクスポート（`anthropicImageTokens`、`resolveAnthropicVisionTier`、ティア上限）。
- **実測済みの本番レンダー構成**: 高密度な1ビットグリフアトラス（アンチエイリアシングなし）、標準ティアのページ — すべての選択は`benchmarks/*/results/`内のベンチマークの根拠に裏付けられています。
- **ベンチマークハーネス**（`benchmarks/`）: billing-sweep（トークン会計）とdensity-frontier（モデル/密度をまたぐ読み取り精度のフロンティア）。API、OpenRouter、Claude Code CLI、またはOmniRoute経由（`--via-omniroute`）で再実行可能。
- **拒否時の再試行**: SSE/JSONスニファーが、モデルがレンダリングされたページを拒否した際に元のリクエストを再送します（キルスイッチ`retryRefusalWithOriginal`）。
- **決定論的なページのためのLRUレンダーキャッシュ**。
- **OmniRouteエンジン**: [OmniRoute](https://github.com/diegosouzapw/OmniRoute)内で`omniglyph`圧縮エンジンとして出荷（単体モードとスタック型パイプライン）、フェイルクローズなゲートと画像対応のトークン会計を備えます。

### 数値（すべて再現可能）

- サンプルUIレンダー: 1015文字 → 438×120 PNG、254 → 84トークン（**66.9%節約**）。
- 標準ページ1568×728 = テキスト量にかかわらず1456画像トークン。
- Claudeは本番密度で高密度1ビットページを100%読み取り。Opus 4.8は10×16で77–87%読み取り。

### 見送った決定（意見ではなく実測に基づく）

- **高解像度ティアは課金の罠**: 1928²ページはWYSIWYGで課金されますが、エンコーダは全解像度を受け取っていません — どちらのティアも標準ページをレンダリングします。
- **GPT-5.5は不採用**: 高密度ストリップの読み取りが0/60、かつテキスト対照群比で完了トークンが約40倍膨張。
- **gpt-4o-miniは画像化しない**（2833/5667トークンという下限のため採算が合わない）。
- **Gemini 2.5-flashは高密度ページで棄権せずに作話**（0/26） — 有償クォータでの再テストを保留中。
