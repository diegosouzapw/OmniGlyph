# OmniGlyph

> LLM向けのコンテキストを画像として扱う圧縮プロキシ。各リクエストのバルクな部分
> （system prompt、ツールのドキュメント、古い履歴、大きなツール出力）を、マシンから
> 送信される前に高密度な1ビットPNGページへ書き換えるローカルプロキシで、プロバイダごとの
> 厳密な課金計算（Anthropic/OpenAI/Gemini）を用い、トークン計算上有利な場合のみ変換します。
> 実測されたエンドツーエンドの請求削減率: 59〜70%。OmniRouteファミリーの一部であり、
> OmniRoute内では`omniglyph`圧縮エンジンとしても出荷されています。

**主な価値:** Anthropic互換の任意のツールを
`ANTHROPIC_BASE_URL=http://127.0.0.1:47821`に向けるだけで、バルクなコンテキストが
テキストトークン（約2文字/トークン）ではなく画像トークン（約19文字/トークン）として課金されます。

**現在のバージョン:** 1.0.0 · **ライセンス:** MIT · **ランタイム:** Node ≥18 および
Cloudflare Workers。

## 仕組み

1. 収益性ゲート: プロバイダの厳密な画像課金を計算し
   （Anthropic: ⌈w/28⌉ × ⌈h/28⌉ + ブロックオーバーヘッド、残差ゼロまで実測済み）、
   トークンが節約できる場合のみ変換します。
2. モデルゲート（フェイルクローズ）: 読み取りベンチマークに合格したモデルのみが画像を
   受け取ります。現在承認されているのは: Claude Fable 5（完全一致100%）。
   証拠付きで遮断されているのは: GPT-5.5（0/60）、Gemini 2.5-flash（作話する）、
   Opus 4.8はオプトインのセーフモード（4倍のグリフ）でのみ。
3. レンダリング: リフロー + 1ビット5×8グリフアトラス → 標準の1568×728 PNGページ
   （高解像度ティアは実測された課金の罠であり、使用されません）。
4. ページをキャッシュフレンドリーな形で元に戻す。バイト単位で正確な値（ハッシュ/ID）は
   テキストとして併走し、読み取りに失敗した場合は棄権します
   （約300件のプローブで無自覚な作話ゼロ）。
5. 拒否時の再試行: モデルがレンダリングされたページを拒否した場合、元のリクエストが
   自動的に再送されます。

## プロジェクト構造

- `src/core/` — 変換パイプライン、厳密な課金計算、レンダラー、LRUレンダーキャッシュ、
  拒否時再試行付きのプロキシ。課金計算はパッケージルートでエクスポートされます。
- `src/node.ts` — ダッシュボード付きのローカルホスト（節約トークン数、並列比較の変換、
  キルスイッチ）。`src/worker.ts` — Cloudflare Workersホスト。
- `benchmarks/` — billing-sweepとdensity-frontierのハーネス。公開されているすべての
  数値は`benchmarks/*/results/`にJSONLの根拠を持ちます。
- `docs/` — architecture/、benchmarks/（手法 + 結果）、ops/（OmniRouteのランブック）、
  i18n/pt-BR/。

## LLM向けの重要な注記

- 厳格なTDDと主張の前の実測: ベンチマークの根拠がない数値は受け入れられず、
  フェイルクローズなゲートはテストを通すために弱めてはいけません。
- 圧縮はリクエスト側のみです — モデルの応答は手を加えずにストリームします。
- プロキシはローカルファーストです: それ以外の送信は第三者へは一切行われません。
