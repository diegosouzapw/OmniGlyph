# 安全政策

## 回報漏洞

請在 GitHub 上開立私人安全通報
(<https://github.com/diegosouzapw/OmniGlyph/security/advisories/new>)
或直接聯絡維護者(diegosouza.pw@outlook.com)。請勿為可被利用的漏洞
開立公開議題。

## 威脅模型(OmniGlyph 是什麼)

OmniGlyph 是位於你的用戶端(例如 Claude Code)與 LLM API 之間的**本機代理**。
依其設計,它會在傳輸過程中看到你所有的工作階段內容與憑證。相應的
安全決策如下:

- **預設綁定於回送位址**(`127.0.0.1`):儀表板沒有身分驗證,
  並會提供已擷取的工作階段內容(圖像的原始文字、遙測資料)。
  `HOST=0.0.0.0` 是明確的選擇性啟用項,會將上述所有內容暴露於網路——
  僅在受信任的網路上使用。
- **憑證**:代理會將用戶端的驗證標頭轉送給上游,且不會保存它們。
  透過環境變數提供的金鑰(`ANTHROPIC_API_KEY` 等)僅存於記憶體中。
- **本機遙測**:`~/.omniglyph/events.jsonl` 保存每個請求的中繼資料
  (token 計數、內容雜湊),以及在 4xx 錯誤時的壓縮內容樣本——
  請將此檔案視為敏感資料。
- **圖像化的內容是有損的**:位元組精確的值(機密、雜湊)絕不能依賴
  圖像讀取;流水線會將其保留為文字,但黃金原則是:不要把機密放進
  LLM 上下文中。
- **供應鏈**:`pnpm-workspace.yaml` 為任何新套件強制實施 3 天的
  `minimumReleaseAge`;核心僅有單一執行時期依賴。

## 支援的版本

僅最新的發行分支(`main` / 最新的 `v1.x`)會收到修復。
