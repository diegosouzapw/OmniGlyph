# Anthropic vision-billing sweep

రెండు ఓపెన్ జ్యామితి ప్రశ్నలను నిర్ణయించే ఉచిత `count_tokens` స్వీప్:

1. **Formula** — API `ceil(w/28) × ceil(h/28)` ప్యాచ్‌లను (ప్రస్తుత
   డాక్స్) బిల్ చేస్తుందా లేదా పదవీ విరమణ చేసిన `w·h/750`నా? ప్రోబ్
   సెట్ ఈ రెండింటినీ రోకు 25–180 టోకెన్‌ల ద్వారా వేరు చేస్తుంది.
2. **Tier** — `claude-fable-5`కు హై-రిజల్యూషన్ క్యాప్‌లు లభిస్తాయా
   (long edge ≤ 2576 px, ≤ 4784 విజువల్ టోకెన్‌లు)? `page-old-1928x1928`
   రో నిర్ణయాత్మకమైనది: కొలిచినది ≈ **4761** అంటే హై-రెస్ WYSIWYG
   (పాత పెద్ద పేజీ నేటి 1568×728 కంటే ఇమేజ్‌కు ~3.3× ఎక్కువ అక్షరాలను
   అదే chars/token వద్ద కలిగి ఉంటుంది); ≈ **1521** అంటే స్టాండర్డ్-టైర్
   resample, మరియు 1568×728 సరైనదిగానే ఉంటుంది.

Context: ప్రస్తుత 1568×728 పేజీ వెనుక ఉన్న 2026-07-01 స్వీప్
(చదవగలిగే స్థాయి ఆడిట్, 2026-07-01) `claude-sonnet-4-5`పై కొలవబడింది
— ఒక స్టాండర్డ్-టైర్ మోడల్ — ప్రొడక్షన్ లక్ష్యం Fable 5, దీన్ని విజన్
డాక్స్ హై-రిజల్యూషన్ టైర్‌లో ఉంచుతాయి. ఆ ఆడిట్ ప్రస్తుత పేజీని 1460
టోకెన్‌లుగా కూడా కొలిచింది: /750 యొక్క 1522 కంటే ప్యాచ్ ఫార్ములా యొక్క
1456కు దగ్గరగా, API ఇప్పటికే ప్యాచ్ బిల్లింగ్‌కు మారిందని సూచిస్తుంది.

## Run

```bash
pnpm run build                              # dist/ prerequisite (like all evals)
node benchmarks/billing-sweep/run.mjs --dry-run   # predictions only, no key, $0

ANTHROPIC_API_KEY=sk-... node benchmarks/billing-sweep/run.mjs \
  --models claude-fable-5,claude-sonnet-4-5 --probe-multi --probe-20plus
```

APIని **నేరుగా** హిట్ చేయాలి — OmniGlyph ప్రాక్సీ ద్వారా ఎప్పుడూ కాదు,
అది బాడీని ట్రాన్స్‌ఫార్మ్ చేస్తుంది. `count_tokens` ఉచితం; పూర్తి
స్వీప్ ~25 రిక్వెస్ట్‌లు చేస్తుంది.

## Reading the output

మోడల్‌కు, ప్రతి ప్రోబ్ రో కొలిచిన ఇమేజ్ టోకెన్‌లను (with-image మైనస్
text-only బేస్‌లైన్) అన్ని నాలుగు ప్రిడిక్షన్‌లకు వ్యతిరేకంగా చూపిస్తుంది
(`patch`/`legacy750` × `standard`/`highres`); సారాంశం పరికల్పనలను
మీన్ అబ్సొల్యూట్ అవశేషం ద్వారా ర్యాంక్ చేస్తుంది. `--probe-multi`
ఇమేజ్‌కు క్యాప్‌ను చెక్ చేస్తుంది (2×1092² ≈ 2×1521); `--probe-20plus`
>20-images నియమాన్ని చెక్ చేస్తుంది (>2000 px సైడ్ తిరస్కరించబడాలి,
resample కాదు). రోలు `results/*.jsonl`లో ల్యాండ్ అవుతాయి; ప్రిడిక్షన్
గణితం `formulas.mjs`లో ఉంది, `tests/billing-sweep-formulas.test.ts`
ద్వారా పిన్ చేయబడింది.
