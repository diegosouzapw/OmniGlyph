# Політика безпеки

## Повідомлення про вразливості

Відкрийте приватний security advisory на GitHub
(<https://github.com/diegosouzapw/OmniGlyph/security/advisories/new>) або
зв'яжіться з мейнтейнером напряму (diegosouza.pw@outlook.com). Не відкривайте
публічний issue для вразливості, яку можна експлуатувати.

## Модель загроз (що таке OmniGlyph)

OmniGlyph — це **локальний проксі** між вашим клієнтом (наприклад,
Claude Code) і API LLM. За задумом він бачить увесь вміст вашої сесії та
ваші облікові дані під час передачі. Відповідні рішення щодо безпеки:

- **За замовчуванням прив'язаний до loopback** (`127.0.0.1`): дашборд не має
  автентифікації і віддає захоплений контекст сесії (вихідний текст
  зображень, телеметрію). `HOST=0.0.0.0` — це явний opt-in, який відкриває
  все це мережі — використовуйте лише в довіреній мережі.
- **Облікові дані**: проксі пересилає заголовки автентифікації клієнта на
  вищий рівень і не зберігає їх. Ключі, надані через env
  (`ANTHROPIC_API_KEY` тощо), залишаються в пам'яті.
- **Локальна телеметрія**: `~/.omniglyph/events.jsonl` містить метадані по
  кожному запиту (кількість токенів, хеші тіла) і, у разі помилок 4xx,
  стиснені зразки тіла — вважайте цей файл чутливим.
- **Перетворений на зображення вміст втратний**: побайтово точні значення
  (секрети, хеші) ніколи не повинні залежати від читання зображень; конвеєр
  зберігає їх як текст, але золоте правило таке: не кладіть секрети в
  контекст LLM.
- **Ланцюг постачання**: `pnpm-workspace.yaml` встановлює `minimumReleaseAge`
  у 3 дні для будь-якого нового пакета; ядро має лише одну runtime-залежність.

## Підтримувані версії

Виправлення отримує лише остання лінія релізу (`main` / найновіша `v1.x`).
