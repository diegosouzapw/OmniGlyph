# Бенчмарки

🌐 Перекладено: [усі мови](../../README.md)

Кожна цифра, яку заявляє OmniGlyph, походить з одного з двох тестових
стендів нижче — їх можна перезапустити, вони детерміновані там, де це
можливо, з сирими підтвердженнями по кожній відповіді в `*/results/*.jsonl`.
Консолідований аналіз: [docs/benchmarks/BENCHMARKS.md](../docs/benchmarks/BENCHMARKS.md).

## Як працює економія (в одній картинці)

Провайдери тарифікують **текст за токеном**, але тарифікують **зображення
за його розмірами** — а не за тим, скільки тексту в нього вміщено. Одна
стандартна сторінка має фіксовану вартість незалежно від того, наскільки
щільний текст:

```
                         ┌─────────────────────────────────────────────┐
28,080 characters   →    │  one 1568 × 728 PNG page   =  1,460 tokens   │
of dense context         └─────────────────────────────────────────────┘
                                          the SAME 1,460 whether the page
                                          holds 200 chars or 28,080
```

Той самий контекст, тарифікований двома способами:

```
as dense TEXT    ██████████████████████████████████████████████  ~14,040 tokens
as ONE IMAGE     █████                                              1,460 tokens
                                                                    ▼
                                                              ~10× fewer tokens
```

Чому зображення перемагає — символів на токен:

```
image (dense page)  ███████████████████████████████████████  19.2 chars/token
dense text          ████                                       ~2  chars/token
```

OmniGlyph робить цю заміну лише тоді, коли точна математика показує, що
вона вигідна, і лише для моделей, які доведено вміють читати сторінку. Два
тестові стенди нижче доводять кожну з цих половин.

## 1. `billing-sweep/` — скільки насправді коштує зображення?

Безкоштовні проби `count_tokens` проти живого API Anthropic, що порівнюють
застарілу формулу `w·h/750` з поточною моделлю патчів 28 px на 11 пробних
геометріях, 2 моделях × 2 тарифах роздільності.

**Результат (2026-07-05): модель патчів підходить із НУЛЬОВИМ залишком у
кожній пробі** — тарифікується = `⌈w/28⌉ × ⌈h/28⌉` після зміни розміру за
тарифом, плюс фіксовані +3/+4 токени на блок зображення. Продакшн-сторінка
(1568×728) коштує рівно 1,460 токенів і несе 28,080 символів ≈ **19.2
символи/токен** проти ~2 символи/токен для щільного тексту.

```
the page as the patch grid the API actually bills:

  ⌈1568 / 28⌉ = 56 patches wide
  ⌈ 728 / 28⌉ = 26 patches tall
  ───────────────────────────────
  56 × 26  = 1,456  + 4 per-block  =  1,460 tokens   (flat, WYSIWYG)
```

```bash
node benchmarks/billing-sweep/run.mjs --dry-run          # predictions only, $0
ANTHROPIC_API_KEY=... node benchmarks/billing-sweep/run.mjs   # live sweep, still $0 (count_tokens is free)
```

## 2. `density-frontier/` — чи може модель насправді ПРОЧИТАТИ це?

Вартість (офлайн, точно) × точність читання (наживо) на різних
конфігураціях рендерингу, геометрії сторінок, атласах гліфів і
провайдерах. Корпус закладає точні рядкові needle-питання (hex ids,
camelCase, послідовності цифр) плюс **дистрактори "майже промах",
побудовані з виміряних пар плутанини гліфів** — тож мовчазна конфабуляція
виявляється, а не просто рахується як помилка. Підрахунок детермінований
(без LLM-судді): `correct` / `abstained` (чесний `ILEGIVEL`) /
`silent_wrong` / `no_answer`.

**Головні результати** (n=30 на плече):

| плече | точних читань | нотатки |
|---|---:|---|
| Fable 5 · стандартна сторінка · атлас 1-bit (продакшн) | **30/30** | нуль помилок, нуль конфабуляцій |
| Fable 5 · стандартна сторінка · атлас AA (старий дефолт) | 25/30 | 5 чесних утримань — чому продакшн перемкнувся на 1-bit |
| Fable 5 · сторінка high-res 1928² | 1–2/30 | тарифікується у 3.3× дорожче, але resample енкодером — пастка тарифікації, не увімкнено |
| Opus 4.8 · гліфи 10×16 | 23–26/30 | безпечний режим за бажанням |
| GPT-5.5 · смуга 768px (обидва атласи) | 0/60 | + ~40× роздування вихідних токенів проти власного текстового контролю (30/30, 62 tok) |
| Gemini 2.5-flash (частково, квота) | 0/26 | конфабулює замість того, щоб утримуватися |

Точність читання наочно — це і **є** намальований fail-closed гейт моделі:

```
Fable 5 · 1-bit page (prod)   ██████████████████████████████  30/30  ✅ approved
Opus 4.8 · 10×16 (safe mode)  ████████████████████████░░░░░░  ~24/30 ⚠️ opt-in
Fable 5 · high-res 1928²      █░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  ~2/30  🚫 billing trap
GPT-5.5 · 768px strip         ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  0/60   ⛔ blocked
Gemini 2.5-flash              ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  0/26   ⛔ confabulates
```

У продакшн йде лише плече ✅. Все, що читається погано, блокується *з
підтвердженням*, а трирівнева оцінка означає, що модель, яка вгадує
неправильно (`silent_wrong`), вважається гіршою за ту, яка чесно
утримується (`ILEGIVEL`).

Три транспорти: прямий API (`ANTHROPIC_API_KEY`/`OPENAI_API_KEY`/`GEMINI_API_KEY`),
OpenRouter (`OPENROUTER_API_KEY`) і `--via-cli` (підписка Claude Code —
$0). Застереження, засвоєне на практиці: посередники (OpenRouter,
інструмент Read у CLI) змінюють розмір великих зображень; лише результати
прямого API авторитетні для читабельності.

```bash
pnpm exec tsx benchmarks/density-frontier/run.ts --dry-run                    # cost table, $0
pnpm exec tsx benchmarks/density-frontier/run.ts --via-cli --sections 30     # via subscription, $0
ANTHROPIC_API_KEY=... pnpm exec tsx benchmarks/density-frontier/run.ts --configs anthropic-std-5x8-1bit
```
