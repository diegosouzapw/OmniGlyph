Консолідований план роботи (2026-07-05) за матеріалами: виміряний billing
sweep, аудит OpenAI/Gemini проти офіційної документації, аналіз суміжних
інструментів, і тестовий стенд density-frontier. Статус кожного пункту:

## Результати A/B 2026-07-05 (через OpenRouter — НЕВИЗНАЧЕНО для геометрії,
дійсно для режимів відмови)

| конфігурація | дослівно | утрим. | відфільтровано | мовчазна помилка |
|---|---|---|---|---|
| fable std 5×8 (AA і 1-bit) | 0/30 | 0 | **30/30 content_filter** | 0 |
| fable hires 5×8 (AA) | 0/30 | **20 ILEGIVEL** | 0 | 5 (2 передбачено) |
| fable hires 5×8 (1-bit) | 0/30 | 20 | 1 | 4 (2 передбачено) |
| opus hires 10×16 | **7/9 прочитано** | 0 | 21 закінчились кредити | 2 (цифра) |

Дійсні висновки: (1) класифікатор (issue #37) є ДОМІНАНТНИМ режимом
відмови для питань транскрипції на стандартній сторінці — 100%
відфільтровано — і не спрацьовує на великій сторінці; формулювання має
значення. (2) Утримання працює: 20× ILEGIVEL проти 5 конфабуляцій на
великій сторінці. (3) Opus за 10×16 читає 78% точно (n=9) проти 0%
історично за 5×8 — перший з перших рук доказ "коліна". (4) Нечитабельність
великої сторінки через OpenRouter натякає на транспортний RESAMPLE
(Bedrock/Vertex стандартний тариф?) — вирішальна гіпотеза для тесту на
прямому API Anthropic; A/B геометрії залишається ВІДКРИТИМ до того часу.
Кредити OpenRouter закінчилися посеред плеча Opus.
