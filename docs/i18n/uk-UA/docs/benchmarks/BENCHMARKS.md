## 1. Тарифікація Anthropic (прямий count_tokens, $0, 11 геометрій × 2 моделі)

Підтверджена формула: `tokens = ceil(w/28) × ceil(h/28)` після зміни
розміру за тарифом, **+3/блок (Fable 5) / +4/блок (Sonnet 4.5)** — НУЛЬОВИЙ
залишок у всіх рядках.

| проба | розміри | Fable 5 (high-res) | Sonnet 4.5 (стандарт) |
|---|---|---:|---:|
| doc anchor | 1092×1092 | 1524 | 1525 |
| doc anchor | 1000×1000 | 1299 | 1300 |
| standard page | 1568×728 | 1459 | 1460 |
| **large page** | **1928×1928** | **4764 (WYSIWYG!)** | 1525 (resample) |
| hi-res ceiling | 1960×1960 | 4764 (clamp) | 1525 |
| hi-res long edge | 2576×1204 | 3959 | 1516 |
| tall strip | 768×1932 | 1935 | 1292 (resample) |
| 2×1092² (multi) | — | 3048 (= 2×1524) | 3049 |
| 21×280² (>20 imgs) | — | 2163 | 2164 |
| 20×280²+2100² | — | 6824 (2100²→downscale, НЕ відхилено в count_tokens) | 3585 |

Похідні рішення (реалізовані): точний гейт по патчах; тариф за моделлю
(Fable/Mythos/Opus 4.8/4.7/Sonnet 5 = high-res); `cols` 313→312.

## 2. Точність читання (density-frontier, needle-питання hex/camelCase/цифра
+ дистрактори)

### Матриця Fable 5 2×2 — через CLI/підписку, n=30/плече, той самий корпус
(~16,6 тис. символів)

| сторінка × атлас | точно | утримання (ILEGIVEL) | мовчазні помилки |
|---|---:|---:|---:|
| standard 1568×728 · **1-bit** | **30/30 (100%)** | 0 | 0 |
| standard 1568×728 · AA | 25/30 (83%) | 5 | 0 |
| high-res 1928×1928 · **1-bit** | 20/30 (67%) | 10 | 0 |
| high-res 1928×1928 · AA | 0/30 | 29 | 1 (передбачено матрицею) |

→ **1-bit > AA на обох сторінках; нуль конфабуляцій у 120 питаннях.**
ЗАСТОСОВАНО: `DENSE_RENDER_STYLE` → `aa:false` (коміт 9a25585).
⚠️ high-res надходить деградованою через resample транспорту (див. H1/H3) —
67% — це підлога, а не стеля.

### Opus 4.8 — через CLI/підписку, n=30/плече

| конфігурація | точно | утримання | помилки |
|---|---:|---:|---:|
| high-res · комірка 10×16 | **26/30 (87%)** | 0 | 4 (цифри) |
| standard · комірка 5×8 | 0/30 | 30 | 0 |

→ Коліно Opus підтверджено власним n (апстрім виміряв 95% за 10×16 з
n=20). "Безпечний режим Opus" життєздатний: 10×16 на великій сторінці ≈
1,7 символу на токен зображення на корпусі тестового стенду.

### Через OpenRouter (той самий корпус/питання) — невизначено щодо
читабельності

| виміряний факт | число |
|---|---|
| content_filter на питаннях транскрипції (стандартні сторінки) | 60/60 (100%) |
| content_filter на high-res сторінках | 5-6/30 (~20%) |
| Fable high-res: утримання + помилки | 20 ILEGIVEL + 5 помилок (2 передбачено) |
| Opus 10×16 (до того, як закінчилися кредити) | 7/9 точно (78%) |
| хибні читання, передбачені матрицею confusability | 4→a, 0→8, S/s регістр |

### Порівняння транспортів (те саме питання, той самий вміст)

| транспорт | фільтр/відмова | велика сторінка читабельна? |
|---|---|---|
| Прямий API (n=9, до того, як закінчилися кредити) | 0 | не тестувалося |
| OpenRouter | ~100% std / ~20% hi-res | ні (підозра: resample) |
| Claude Code CLI (підписка) | 0 content_filter; ~50% великих пакетів зависали (вирішено пакетами по 10 + повтор) | ні (підозра: Read змінює розмір) |

## ДОДАТОК 2026-07-06 (2) — Батарея GPT-5.5 через прямий API: H7 закрито

залишається неперевіреним (обліковий запис не має доступу). Майбутнє
(H12): гейт GPT повинен моделювати роздування виводу, а не лише токени
промпту.

## ДОДАТОК 2026-07-06 (3) — Gemini 2.5-flash (ЧАСТКОВО: квота безкоштовного
тарифу вичерпалась посеред прогону)

З ~26 відповідей-зображень, що пройшли до вичерпання квоти: **0 правильних,
1 утримання, ~25 КОНФАБУЛЯЦІЙ** — і це не плутанина гліфів: це випадкові
цифри (`indexLedgerInd → 0040375615`), тобто енкодер бачить майже нічого
за тестованих щільностей (нативний tile 42 символи/токен і MEDIUM flat), а
2.5-flash ВИГАДУЄ замість того, щоб утримуватися (ігнорує інструкцію
ILEGIVEL). Текстовий контроль: 3/3 на тих, що пройшли. Без роздування
виводу (6-28 токенів/відповідь).

Попередній сигнал: H5/H6 схиляються до НІ на 2.5-flash, з режимом відмови
ГІРШИМ, ніж у GPT (мовчазна конфабуляція замість утримання) — Gemini
вимагав би додаткових запобіжників у проксі. Очікує закриття: повторний
запуск з платною квотою або в інший день, і тест gemini-2.5-pro (flash —
найслабший читач у родині). Сторінка з нативним tile все ще має найкраще
ДОКУМЕНТОВАНЕ співвідношення (42,3 символи/токен); саме читабельність
викликає сумніви.

Примітка щодо вартості: часткові сторінки (остання в корпусі) тарифікуються
погано в режимі tile (менша висота → менша одиниця обрізки → більше tiles)
— доповнення останньої сторінки до висоти 1152px є обов'язковою
оптимізацією, якщо Gemini буде включено.
