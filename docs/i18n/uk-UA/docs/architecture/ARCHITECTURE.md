# Архітектура

Карта кодової бази на одну сторінку.

## Конвеєр запиту

```
клієнт (Claude Code / будь-який SDK)
  │  ANTHROPIC_BASE_URL=http://127.0.0.1:47821
  ▼
src/node.ts · src/worker.ts        хости (node:http / Cloudflare Workers)
  ▼
src/core/proxy.ts                  єдиний обробник fetch за Web-стандартом:
  │                                маршрутизація, наскрізна автентифікація,
  │                                контрфактичний count_tokens, події
  │                                usage/телеметрії
  ▼
src/core/transform.ts              КОНВЕЄР (шлях Anthropic):
  │   1. розбір тіла, визначення тарифу зору за моделлю
  │   2. гейт прибутковості — точна вартість зображення проти вартості тексту
  │   3. конвертація: статичний slab · великі tool_results · згорнута історія
  │   4. зшивання назад зі збереженням якорів cache_control клієнта
  ▼
upstream API (api.anthropic.com / api.openai.com)
```

## Тарифікація (точна, виміряна)

| модуль | провайдер | модель |
|---|---|---|
| `src/core/anthropic-vision.ts` | Anthropic | патчі 28 px + 4/блок, ліміти зміни розміру за тарифом; геометрія сторінки (обидва тарифи рендерять стандартну сторінку 1568×728 — тариф високої роздільності є пасткою тарифікації, див. [BENCHMARKS](../benchmarks/BENCHMARKS.md)) |
| `src/core/gpt-model-profiles.ts` + `openai.ts` | OpenAI | режими patch/tile за моделлю, `detail` за профілем, геометрія смуги |
| `src/core/gemini-model-profiles.ts` | Google | формула tile (одиниця обрізки `floor(min/1.5)`) + фіксовані вартості `media_resolution` |

## Рендеринг

- `src/core/render.ts` — текст → PNG через запечений атлас гліфів
  (Spleen 5×8 + резервний Unifont), reflow з сентинелами нового рядка `↵`,
  1-бітний атлас у продакшні (виміряно краще, ніж AA, на Fable).
- `src/core/render-cache.ts` — LRU-мемоізація детермінованих рендерів
  (статичний slab + заморожені фрагменти історії інакше перерендерювалися б
  при кожному запиті).
- `src/core/history.ts` — згортає старі ходи в append-only заморожені
  фрагменти зображення, які лишаються побайтово ідентичними, щоб кешування
  промптів продовжувало спрацьовувати.
- `src/core/png.ts` — мінімальний детермінований PNG-енкодер (без нативних
  залежностей).

## Захисні бар'єри

- Allowlist моделей (`src/core/applicability.ts`): лише моделі, що пройшли
  бенчмарк читання, отримують зображення; усе інше проходить побайтово
  ідентичним.
- Побайтово точні значення (SHA, id) їдуть як текст у fact sheet поруч із
  зображенням (`src/core/factsheet.ts`); відновлювані оригінали через
  `emitRecoverable`.
- Нативні типізовані інструменти (`type !== 'custom'`) ніколи не
  переписуються (охорона 400).
