# बदल-सूची (Changelog)

स्वरूप: [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) · सिमँटिक व्हर्जनिंग.

## [1.0.0] — 2026-07-07

पहिली सार्वजनिक रिलीज.

### उत्पादन

- **संदर्भ-प्रतिमारूप संकुचन प्रॉक्सी**: प्रत्येक LLM विनंतीच्या दाट भागांना
  (सिस्टम प्रॉम्प्ट, टूल डॉक्स, जुना इतिहास, मोठी टूल आउटपुट्स) तुमच्या मशीनवरून
  बाहेर जाण्यापूर्वी घन 1-बिट PNG पानांमध्ये पुनर्लेखन करतो. लोकल Node सर्व्हर
  आणि Cloudflare Workers होस्ट.
- **प्रति-प्रदाता अचूक बिलिंग गणित** (`src/core/`): Anthropic 28px पॅचेस +
  3–4 टोकन्स/ब्लॉक ओव्हरहेड (स्वतःचा स्वीप, शून्य अवशेष), OpenAI आणि Gemini
  सूत्रे अधिकृत दस्तऐवजांविरुद्ध तपासलेली. पॅकेज रूटवर एक्सपोर्ट केलेली
  (`anthropicImageTokens`, `resolveAnthropicVisionTier`, tier caps).
- **मोजलेला उत्पादन रेंडर कॉन्फिग**: घन 1-बिट ग्लिफ अॅटलास (अँटी-अलायझिंगशिवाय),
  स्टँडर्ड-टियर पाने — प्रत्येक निवडीला `benchmarks/*/results/` मध्ये बेंचमार्क
  पावतीचा आधार.
- **बेंचमार्क हार्नेस** (`benchmarks/`): billing-sweep (टोकन अकाउंटिंग) आणि
  density-frontier (मॉडेल्स/डेन्सिटींमधील वाचन-अचूकता सीमा), API, OpenRouter,
  Claude Code CLI, किंवा OmniRoute (`--via-omniroute`) द्वारे पुन्हा-चालवण्यायोग्य.
- **नकार पुनर्प्रयत्न (Refusal retry)**: जेव्हा मॉडेल रेंडर केलेले पान नाकारतो
  तेव्हा SSE/JSON स्निफर मूळ विनंती पुन्हा प्ले करतो (किल स्विच
  `retryRefusalWithOriginal`).
- निश्चयात्मक पानांसाठी **LRU रेंडर कॅशे**.
- **OmniRoute इंजिन**: [OmniRoute](https://github.com/diegosouzapw/OmniRoute)
  मध्ये `omniglyph` संकुचन इंजिन म्हणून जहाज होतो (सिंगल मोड आणि स्टॅक्ड
  पाइपलाइन दोन्ही), फेल-क्लोज्ड गेट्स आणि इमेज-जागृत टोकन अकाउंटिंगसह.

### आकडे (सर्व पुनरुत्पादनीय)

- नमुना UI रेंडर: 1015 अक्षरे → 438×120 PNG, 254 → 84 टोकन्स (**66.9% बचत**).
- स्टँडर्ड पान 1568×728 = 1456 इमेज टोकन्स, त्यात किती मजकूर आहे यावर अवलंबून
  नाही.
- Claude उत्पादन डेन्सिटीवर घन 1-बिट पाने 100% वाचतो; Opus 4.8 10×16 वर
  77–87% वाचतो.

### नकारात्मक निर्णय (मोजलेले, मते नाहीत)

- **उच्च-रिझोल्यूशन टियर हा एक बिलिंग सापळा आहे**: 1928² पानाचे शुल्क
  WYSIWYG प्रमाणे आकारले जाते पण एन्कोडरला पूर्ण रिझोल्यूशन मिळत नाही — दोन्ही
  टियर स्टँडर्ड पाने रेंडर करतात.
- **GPT-5.5 नाकारले**: घन स्ट्रिपचे 0/60 वाचन आणि मजकूर नियंत्रणाच्या तुलनेत
  ~40× कम्प्लीशन फुगवटा.
- **gpt-4o-mini कधीच इमेज केले नाही** (2833/5667 टोकन तळामुळे ते फायदेशीर
  ठरत नाही).
- **Gemini 2.5-flash घन पानांवर थांबण्याऐवजी गोंधळ निर्माण करतो**
  (0/26) — सशुल्क-कोटा पुनर्चाचणीच्या प्रतीक्षेत.
