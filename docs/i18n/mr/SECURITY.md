## धोका मॉडेल (OmniGlyph काय आहे)

OmniGlyph हा तुमचा क्लायंट (उदा. Claude Code) आणि LLM APIs यांच्यातील एक
**लोकल प्रॉक्सी** आहे. डिझाइननुसार तो तुमच्या सत्रातील संपूर्ण मजकूर आणि
संक्रमणादरम्यान तुमची क्रेडेन्शियल्स पाहतो. संबंधित सुरक्षा निर्णय:

- **डीफॉल्टनुसार loopback ला बांधलेला** (`127.0.0.1`): डॅशबोर्डमध्ये
  कोणतेही प्रमाणीकरण नाही आणि तो कॅप्चर केलेला सत्र संदर्भ (प्रतिमांचा
  स्रोत मजकूर, टेलिमेट्री) सर्व्ह करतो. `HOST=0.0.0.0` हा स्पष्ट ऑप्ट-इन
  आहे आणि ते सर्व नेटवर्कला उघड करतो — फक्त विश्वासार्ह नेटवर्कवरच वापरा.
- **क्रेडेन्शियल्स**: प्रॉक्सी क्लायंटचे auth हेडर्स अपस्ट्रीमला फॉरवर्ड
  करतो आणि ते साठवत नाही. env द्वारे पुरवलेल्या की (`ANTHROPIC_API_KEY`
  इत्यादी) मेमरीमध्येच राहतात.
- **लोकल टेलिमेट्री**: `~/.omniglyph/events.jsonl` मध्ये प्रति-विनंती
  मेटाडेटा (टोकन गणना, बॉडी हॅशेस) आणि, 4xx त्रुटींवर, संकुचित बॉडी नमुने
  असतात — या फाइलला संवेदनशील माना.
- **इमेज केलेला मजकूर नुकसानकारी आहे**: बाइट-अचूक मूल्ये (सिक्रेट्स, हॅशेस)
  कधीच इमेज वाचनावर अवलंबून नसावीत; पाइपलाइन त्यांना मजकूर म्हणून ठेवते,
  पण सुवर्ण नियम आहे: LLM संदर्भात सिक्रेट्स ठेवू नका.
- **सप्लाय चेन**: `pnpm-workspace.yaml` कोणत्याही नव्या पॅकेजसाठी
  3 दिवसांचा `minimumReleaseAge` सक्तीने लागू करतो; कोरला एकच रनटाइम
  डिपेंडन्सी आहे.
