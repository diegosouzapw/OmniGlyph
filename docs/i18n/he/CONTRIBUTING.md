# תרומה ל-OmniGlyph

תודה על העניין! לפרויקט הזה יש שני כללי תרבות שאינם ניתנים למשא ומתן —
הם הסיבה שכל מספר ב-README ניתן לאמון.

## כלל 1 — TDD קפדני

כל קוד ייצור נולד מבדיקה שנכשלה קודם:

1. כתבו את הבדיקה ו**צפו בה נכשלת מהסיבה הנכונה**.
2. כתבו את המינימום כדי שתעבור.
3. בצעו ריפקטור תוך שמירה על ירוק.

הרף המלא הוא: `pnpm run typecheck && pnpm test && pnpm run build` — כל
השלושה, תמיד (בדיקת קישורי התיעוד ושומר המיתוג-מחדש רצים בתוך `pnpm test`
דרך `tests/docs-integrity.test.ts`).

## כלל 2 — מדידה לפני טענות

שום שינוי לגאומטריה, לאטלס, לנוסחת החיוב, או להיקף המודלים לא נכנס בלי
מספר נמדד. המאגר בנוי סביב המשמעת הזו:

- עלות חיוב ← הוכיחו עם `benchmarks/billing-sweep/` (`count_tokens` חינמי;
  שארית צפויה: אפס).
- קריאוּת ← הוכיחו עם `benchmarks/density-frontier/` (n≥30 לכל זרוע,
  ניקוד דטרמיניסטי, קבלות JSONL שמוגשות ל-`benchmarks/*/results/`).
- רף הקבלה לשינוי ברירת מחדל בייצור: gist == בקרת טקסט
  **וגם** אפס שגיאות מחרוזת-מדויקת שקטות **וגם** חיסכון חיובי.

השערות ללא מספרים הולכות ל-`docs/ROADMAP.md` כהשערות — לעולם לא ל-README
כעובדות. שני רעיונות "מובנים מאליהם" כבר הופרכו עם נתונים
(העמוד ברזולוציה גבוהה, האטלס עם anti-aliasing); התהליך עובד.

## הגדרה

```bash
pnpm install
pnpm test              # full suite, ~40–90s
pnpm run dev:node      # local proxy in watch mode
```

Node ≥18 (CI בודק 20/22/24), pnpm 10 (מוצמד על ידי `packageManager` ב-
package.json).

## מבנה

| תיקייה | כלל |
|---|---|
| `src/core/` | אגנוסטי לזמן ריצה (רק Web APIs — רץ על Node ו-Workers) |
| `src/node.ts` / `src/worker.ts` | אינסטלציית מארח בלבד |
| `benchmarks/` | רתמות ניתנות להרצה מחדש; תוצאות JSONL הן קבלות, מוגשות |
| `docs/` | benchmarks/ (מספרים), architecture/ (מפה), ROADMAP (השערות), ops/ (OmniRoute) |

## קומיטים ו-PR-ים

- קומיטים קונבנציונליים (`feat:`, `fix:`, `perf:`, `docs:`, `refactor:`,
  `test:`, `chore:`), עם גוף שמסביר את ה*מדוע* עם המספרים הרלוונטיים.
- PR-ים קטנים וממוקדים; שינויי התנהגות מגיעים עם הבדיקה שמצמידה אותם ו,
  כשרלוונטי, הבנצ'מרק שמצדיק אותם.
- אל תכתבו מחדש בלוקי `cache_control` של הלקוח, אל תוסיפו תלויות זמן ריצה
  בלי דיון (הליבה קלת-תלויות בכוונה), אל תשתמשו ב-
  `Math.random`/חותמות זמן בנתיבי הרינדור (דטרמיניזם הוא אינווריאנט קשיח,
  נבדק על ידי זהות-בייטים).
